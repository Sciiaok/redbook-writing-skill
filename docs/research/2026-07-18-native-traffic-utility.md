## 先给结论

这不是一轮新的站内采集，而是对 2026-07-17 已有观察的证据审计。委派环境没有可用的已登录浏览器绑定，因此没有刷新页面、没有切换别的登录方式，也没有使用脚本绕过访问。原始来源及哈希写在私有清单：

`/Users/hermione/Documents/redbook/research/xiaohongshu/_style_library/native-utility/manifest.json`

五组内容都能产生值得继续验证的“任务—载体”假设，但目前 **0/5 满足可晋级的高低配对证据合同**。不能把它们写成“爆款公式”，也不能把公开赞、藏、评改称流量、曝光或推荐权重。

当前最有价值的不是某种配色，而是五个待验证结构：

1. 决策题用明确比较轴压缩选择，而不是做装饰性产品海报。
2. 高意图清单把答案做成可扫描、可回看的工作表。
3. 截图教程让每页只承担一个路径、一个动作位置和一个结果。
4. 前后变化内容同时提供证据差、过程和具体代价。
5. 复杂解释先证明压缩范围，再把封面与内页分工。

这些都只是 `candidate mechanism`。要成为 Skill 的生产资产，仍需同账号、相近时间、同 carrier、同 primary job 的普通/低帖，以及可重算基线与完整逐页记录。

## 审计口径

- `public_proxy`：仅指页面公开显示的赞、藏、评，不代表曝光、触达、CTR、停留或搜索流量。
- `matched control`：至少同账号、相近账号阶段、同 carrier、同 primary job；目标帖不得进入基线成员。
- `complete page observation`：从封面到末页逐页记录；只看封面、前三页或前两页均不算完整。
- `feature contrast`：只有高帖与低帖的 shared constants 可核对后，差异项才可进入候选机制；不能把高帖特征直接叫差异。
- `baseline receipt`：逐条保存成员 note ID、日期、原始 UI 指标、是否置顶、carrier、primary job、可比性和排除原因。只有一个中位数不算可重算。
- `rights=reference_only_no_reuse`：可抽象研究结构，不复制图片、版式、原句、人物、品牌资产或独特表达。

## 跨样本可保留的生产假设

以下是五组共同指向、但仍需新样本验证的生产假设：

| 假设 | 可执行含义 | 不能说 |
| --- | --- | --- |
| 任务承诺先于装饰 | 封面先让用户认出要解决的任务：选择、检查、操作、验证或理解 | 某类封面必然提高点击 |
| 页面承担单一认知动作 | 一页只做比较、定位、证明、解释或总结中的一件事 | 页数越多/越少越好 |
| 可保存来自未来复用价值 | 给条件、步骤、判断轴、成本或答案地图，并标答案位置 | 收藏权重更高、收藏会触发推荐 |
| 可信感来自证据链 | 原生截图、真实 before/after、来源范围、具体成本各自证明不同事实 | “像真人”“不精致”一定有流量 |
| 封面和内页可分工 | 封面负责确认任务/缺口，内页负责扫描与兑现，不必保持海报式统一 | 深色封面 + 白底内页是通用模板 |

这些假设与小红书公开技术材料能确认的多模态理解、搜索相关性和漏斗分层相容，但站内公开互动样本不能证明推荐因果。

## 机器补采合同

每个候选再次采集时，只有同时满足以下条件，才允许从 `research_lead` 升为 paired candidate：

1. 目标帖：完整 note ID、平台账号 ID、稳定 URL、精确采集时间、原始 UI 指标字符串、发布日期、置顶/合作/投放可见状态。
2. 完整内容：封面到末页逐页功能、OCR 摘要、caption 结构、每页私有截图哈希；缺页即停止晋级。
3. 基线：目标排除后至少 5 条可比成员；逐条保存 ID、日期、指标、carrier、primary job、页数和排除理由。
4. 低帖：至少 1 条同账号、相近时间、同 carrier、同 primary job 的普通/低帖，并完整逐页采集。
5. 配对：先写 shared constants，再写 feature contrast；高帖独有特征不得自动成为机制。
6. 指标：公开赞藏评只能写 `public_proxy`；没有一方后台曝光、CTR、停留、搜索入口或分享数据时，`traffic_verdict=unavailable`。
7. 权利：默认 `reference_only_no_reuse`；不得复制图片、原句、独特布局、人物和品牌资产。
8. 复现：同一候选机制至少需要第二个独立账号及其低帖对照，才能考虑跨类目迁移。

JSONL 同步文件：`docs/research/2026-07-18-native-traffic-utility.jsonl`。其中每条记录都把未知字段保留为 `null` 或明确的 gap code，而不是补写想象值。

## 对 Skill 的直接含义

这五组当前应进入 `research lead + gap map`，不应进入默认风格 starter。生成实用型内容时可以调用它们提出版式问题，但必须 fail closed：

- 没有完整高低配对，不得声称“爆款风格”。
- 没有完整页面，不得让模型凭前两页补出“原生轮播模板”。
- 没有目标任务匹配，不得把比较板用于关系故事、把政策编辑用于生活 PLOG。
- 没有真实截图、过程、费用或来源，不得用 AI 画面冒充证据。
- 没有后台数据，公开互动只能帮助挑研究对象，不能给出流量承诺。
