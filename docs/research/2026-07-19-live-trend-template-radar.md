# 2026-07-19 小红书近期模板雷达：现场线索与方法来源

## 这批资料为什么保存

这不是“当前必拍榜”，而是本次新增 [趋势模板雷达](../../redbook-writing/references/trend-template-radar.md) 的现场输入。它记录了怎样从一个已知例子进入精确短语、二创链、评论作业、教程/生成器和普通样本，并据此发现旧 Skill 缺少的字段。

采集日：2026-07-19。入口：登录状态下的小红书公开搜索/详情页。页面显示值只按当时可见文本保存；搜索卡片与详情页口径可能不同。独立复核时，7 个详情 URL 已返回“当前笔记暂时无法浏览”，且本轮没有保存可公开提交的页面快照，因此这些行只保留为 `session_visible_at_capture` 的方法形成线索，不能承担可重放来源。没有后台 impressions/reach，也没有完成所有账号的近期基线，因此全部证据上限为 `public_proxy_association`，多数仍只是 observed 线索。

## 本轮发现路径

```text
用户给出的“从夯到拉”
→ 站内搜精确短语、模板、当前结果和类目替换
→ 比较旧高互动样本与当前普通样本
→ 从标题/评论派生“全网都在拍、交作业、挑战、教程”
→ 打开代表帖，看镜头、动作、评论视频回复和求模板语言
→ 用公开搜索引擎查精确短语的跨平台出现时间
→ 将“排名轴、平凡照片反差、禁拍提示二创、手部转场、怼脸教程、随机挑战”拆成不同模板类型
```

真正有用的不是多搜几个“爆款”大词，而是：发现一个可辨识短语后，追它的别名、BGM/话题、教程、评论作业、不同类目复刻和普通/低表现版本。

## 从这批线索提炼出的关键字段

旧 Skill 可以保存 post、账号基线和风格观察，但无法回答：

- 这篇是母题、直接复刻、类目迁移、反讽、教程、生成器还是同名无关内容？
- 同一个模板的固定槽位、可替换槽位、镜头语法和参与规则是什么？
- 当前是只观察到一篇、已跨账号复刻、仍在上升、已经成熟，还是开始疲劳？
- 生成时应该直接拍、改后拍、观察还是不追？

因此新增两层数据：逐帖 `trend-template-samples.csv` 保存证据角色，候选 `trend-template-candidates.jsonl` 保存家族、四段语法、生命周期、决策和素材门。

## 边界与下一次补采

1. 本批是方法形成样本，不是完整当期模板榜；部分线索只有单篇或搜索卡片，详情 URL 在独立复核时已不可重放。
2. 没有系统完成每个候选的综合/热门/最新、同账号基线和低表现对照。
3. 评论区视频作业需要逐一打开、去重并确认是否独立创作者。
4. 没有保存 URL/快照的跨平台来源线索已从结构化证据中排除；以后必须当场写 source log。
5. 下次 refresh 应先补 `从夯到拉` 的当前普通/低对照、军旅提示二创的独立账号链，以及“平凡照片反差”的小红书站内变体。

这批证据没有进入可直接生成的 starter。调用模板前仍须按雷达完成当期 refresh、素材、版权、真实性与任务适配门。
