# 一次性审美提示词：公开经验源与反例审计

> 采集日期：2026-07-18
> 目的：给探索原型提供比“高级感、活人感、小红书风”更具体的输入，同时阻止课程口诀、模板工具营销和单篇案例被固化成通用爆款样式。
> 证据边界：本文件只产生 `curated_bootstrap / candidate_only / prototype_only` 输入；不产生 published style rule、starter pack 或流量因果。

## 先给结论

公开经验并没有支持一种跨类目的“小红书爆款审美”。相反，同一批资料同时出现精致大图、近人近物、纯字卡、对比图、真实随手拍、严格网格和模板卡片。真正可迁移的不是某个滤镜或字体，而是四个判断：

1. 首屏让当前任务最关键的证据先到，而不是先套一种风格。
2. 画面有明确阅读顺序；“视觉冲击”必须能说明用户为什么点开。
3. 真实素材承担证明工作，装饰不冒充经历、结果或权威。
4. 审美方向必须与同任务、同载体、同素材条件的站内样本对照；公开课程只能给候选变量。

这也是为什么一次性 prompt 只能用于探索：它可以帮助生成一个可看的方向，却不能替代当期小红书样本、逐页观察和发布后的本账号数据。

## 冲突不是噪声，而是选择条件

| 表面口诀 | 资料中的反证 | 进入 Skill 的处理 |
| --- | --- | --- |
| 高颜值实拍才有点击 | 历史商业榜里也有大字信息图；纯标题和比较图同样出现在热门 carrier 中 | 先按 `primary_job` 选择“氛围证据、工作物、比较或命题”，不设默认美图 |
| 大字封面就是爆款 | 数据切片只覆盖特定时期和商业 TOP 样本；同一账号高低帖可共享大字 | 大字只可标 `task_fit` 或 `series_constant`，必须写清它压缩的具体问题 |
| 原生感就是低清随手拍 | 高互动案例的关键是可见现象与制作过程，不是低画质 | 保留真实状态；禁止主动加噪点、假手机 UI、错字或伪生活痕迹 |
| 模板能稳定提高点击 | 模板工具本身有商业激励，且固定比例/配色缺少样本口径 | 模板最多提供探索构图；最终颜色、字形、裁切和密度来自 exact published binding |
| 所有内容都用痛点—方法—互动 | 搜索回答、关系档案、观点、比较、权威说明的阅读任务不同 | 结构由 `primary_job × carrier` 决定；泛化 AI 提示词进入 anti-pattern 库 |
| 活人感就是像素人 | “用户痕迹”来自真实素材、利益关系和参与过程 | 保存 provenance/授权/披露；不伪造真人故事、聊天或自购体验 |

## 可进入一次性 prompt 的 8 个模块

这些模块不是独立母版。每次只选 1 个主模块，必要时加 1 个不冲突的辅助模块；先填写真实素材 ID，再生成 `prototype_only`。

### AP01｜真实工作物先行

- 适用：`search_answer / explain / decision_support`；截图教程、清单、比较、正式解释。
- Prompt 子句：让用户未来会实际查看、填写、比较或操作的工作物承担首屏主体；只用一层标注建立下一步，不把工作物缩成企业海报里的装饰缩略图。内页一页解决一个定位或判断，来源与边界保持可读。
- 禁用：没有真实截图/文件/比较协议；工作物涉及未脱敏隐私；需要 AI 生成证据。
- 来源：`APS01, APS02, APS05`；站内候选 `O-XHS-004/005/006/009`。

### AP02｜真实材质占满画面

- 适用：`trust_build / decision_support`；美妆、食品、器物、服饰、真实使用体验。
- Prompt 子句：让皮肤、质地、磨损、到手状态或人与物的比例成为最大信息面；保留能解释体验的环境光与不完美，只在自然留白放一个结论锚点。图片证明状态，正文解释取舍与不适用条件。
- 禁用：无自有/授权实拍；功效前后无证据；用 AI 手模、皮肤或产品替代实物。
- 来源：`APS03, APS07`；站内候选 `O-XHS-007/010`。

### AP03｜单一真实场景只做在场锚点

- 适用：`relationship_build`；真实体验引出的关系或生活观点。
- Prompt 子句：只用一张本人或已授权的真实环境画面证明“我在这里经历过这件事”；画面保留可核验服务、物件或环境细节，不加大字总结、品牌 KV 或虚构氛围。观点由 caption 里的细节、转折和边界完成，场景不替观点作因果证明。
- 禁用：无本人/授权场景；AI 酒店、人物或房间冒充经历；首图大字讲完观点；无披露植入。
- 来源：`APS05, APS07`；站内候选 `O-XHS-010`。

`APS03` 提供的“可见现象 + 问题缺口”仍保留为研究线索，但当前没有满足 exact scope、逐页 observation 与反例要求的可选 cell，因此没有塞进静态 selector。

### AP04｜跨时间档案保留差异

- 适用：`relationship_build / trust_build`；成长、关系、长期改造、项目复盘。
- Prompt 子句：保留不同年份、设备、光线与裁切的真实差异；不要统一电影滤镜、拍立得边框或卡片底。每页只有一个时间节点和一个“它证明了什么变化”的短标记，caption 再完成因果边界与开放结尾。
- 禁用：没有本人/授权档案；用 AI 人像补齐共同经历；素材顺序不能核实。
- 来源：`APS07`；站内候选 `O-XHS-002/003/012`。

### AP05｜正式编辑，不做企业 KV

- 适用：`search_answer / authority_statement / explain`；政策、法律、研究、复杂知识。
- Prompt 子句：封面只压缩问题、范围与可信承诺；第二页先给完整答案地图，之后用稳定层级逐步展开。正式感来自来源、层级和扫描效率，不来自深蓝渐变、金色光效、图标矩阵或每页同一圆角卡。
- 禁用：事实未核验；来源与结论分离；脚注不可读；用“专业蓝”替代任务证据。
- 来源：`APS01, APS05`；站内候选 `O-XHS-005/009`。

### AP06｜单命题字卡 + 场景兑现

- 适用：`feed_stop / authority_statement`；观点、系列代理、可站队命题。
- Prompt 子句：首图只保留一个能被识别和反驳的命题，系列标记退到次级；不要用贴纸、胶带和装饰手写制造“原生”。正文必须用一个真实场景、选择和后果推进，不能换词复述字卡。
- 禁用：复制他人代理 IP、身体部位、原句或版式；假投稿/假聊天；命题无正文证据。
- 来源：`APS01, APS02, APS06`；站内候选 `O-XHS-011`。

### AP07｜聊天界面只承担推进

- 适用：`relationship_build / feed_stop`；本人聊天、书面授权匿名改编或显著虚构演绎。
- Prompt 子句：先明确来源状态，再只保留改变关系或判断的关键来回；系统界面负责时间、发言者与推进，不叠加便签、旁白卡和伪手写。每一屏必须产生新信息或改变读者判断。
- 禁用：伪造真人投稿；未经授权原聊天；把 AI 写的台词包装成真实截图。
- 来源：`APS06, APS07`；站内候选 `O-XHS-001`。

### AP08｜前后变化必须带代价

- 适用：`trust_build / decision_support`；改造、训练、修复、流程优化。
- Prompt 子句：首屏让真实 before 和目标差先出现；after 可以延迟，但中间每页都给动作或选择证据。结果页必须一起给时间、成本、失败或维护代价，避免只用统一滤镜制造变化。
- 禁用：角度/光线造成虚假差异；没有过程；结果来自生成图；成本由模型估算。
- 来源：`APS03, APS05`；站内候选 `O-XHS-008`。

## 使用与停止条件

```text
exact published style binding 存在？
├─ 是 → binding 决定颜色、字形、裁切、密度和标注语法；本文件只补页面任务
└─ 否 → 仅在显式 exploration 中选 1 个 AP 模块做 prototype_only
         ├─ required material / provenance / rights 齐全 → 生成一个可查看粗原型
         └─ 任一缺失 → prototype_gap / brief_only，不用 AI 补证据
```

连续两次整体评价“像 PPT、像 AI、方向不对”时，不再微调字体或颜色。重新选择目标样本、主模块或真实素材，三者至少改变两项。

## 尚未闭合的证据

- APS01/APS02 是历史第三方榜单，不能说明 2026 feed 机制；只保留为候选变量和历史反例。
- APS04/APS05/APS06 有强烈教学或工具营销激励，不能用于 performance 晋级。
- 8 个模块都需要在当前类目补同任务高/普通/低帖与逐页 hash；跨两个账号、两个内容簇的 typed observation 闭合后，才允许进入 style archetype review。
- 当前公开资产没有第一方 exposure，因此本文件的 prompt 不得附带“高点击率”“爆款”“流量验证”等状态。
