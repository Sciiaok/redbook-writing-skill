# 小红书站内风格观察：高表现、普通对照与适用边界

快照日期：2026-07-17

采集界面：已登录的小红书 Web 搜索页、笔记详情页与作者主页。
用途：为一次性审美提示词和后续风格库提供当前站内线索。本文只记录可观察关联，不声称平台因果，也不把第三方图片或长段原文提交到仓库。

## 口径

- 搜索排序、高赞绝对值和首页位置都不能单独定义“爆款”。
- 优先比较同一作者主页中可见、非置顶、同指标的帖子；目标帖从基线成员中排除。
- `高表现候选` 表示当前快照中相对异常，不等于已经达到 Skill 的 `supported/reusable` 门槛。
- 万级数字为界面显示的近似值，不伪装成精确计数。
- 内容年龄、置顶、付费、热点、账号增长阶段或主页只展示部分帖子时，必须保留限制。

## 逐条观察

### O-XHS-001｜聊天界面直接承担故事

- 查询 / 入口：`聊天记录 故事`，Web 综合搜索。
- 笔记：`《创飞相亲对象的一天》`；作者 `今天也想创飞所有人`；note ID 仅在本轮界面记到前缀 `6a521523`，再次入正式 SQLite 前必须补全身份。
- 可见表现：约 1.7 万赞、161 收藏、492 评论。
- 主页对照：30 条可见帖子互动中位数约 2756.5，目标约 6.17 倍；同一标题/结构的其他帖子从 57 到 5.2 万不等。
- 视觉机制：灰色聊天背景，大面积系统原生白/绿气泡和留白；没有额外卡片、贴纸或品牌 KV。
- 文字推进：开场钩子 → 拒绝/冲突 → 新信息 → punchline；界面顺序本身就是分页节奏。
- 可迁移假设：当信息天然是对话时，让真实界面成为 carrier，避免把聊天再包装进 PPT 卡片。
- 反证/边界：同一载体的表现跨度极大；不能把聊天截图本身写成流量规则，也不能编造“真人故事”或伪造私聊。
- 当前资格：`candidate`；身份未补全，不能充当静态 starter 的 `observed_association`。

### O-XHS-002｜纪念日实拍只留一个身份锚点

- 查询 / 入口：关系/纪念日相关搜索。
- 笔记：`10th anniversary | 十周年`；作者 `柔妹`；note ID 前缀 `67a4d5ca`，正式入库前补全。
- 可见表现：5677 赞、4481 收藏、149 评论。
- 视觉机制：真实牵手近景，偏暗暖色自然光；封面只保留稀疏的“10周年 / 结婚纪念日”字样。
- 文案机制：用 15 年恋爱、10 年婚姻和当天的小事提供具体时间与生活证据；稀疏 emoji 和换行，不写宏大鸡汤。
- 可迁移假设：关系内容可由一个真实动作 + 明确时间锚点建立可信度，正文补足生活细节。
- 反证/边界：尚无同账号基线；不能因为照片“氛围好”就宣称高表现，也不能用 AI 合成照片冒充经历。
- 当前资格：`candidate`。

### O-XHS-003｜PLOG 的信息密度来自照片变化，不来自统一卡片

- 可迁移假设：PLOG 的节奏来自场景切换与尺度变化，不能把每页塞进同一圆角卡片框。
- 反证/边界：未完成账号基线与内容年龄控制；不得把任意照片拼贴都称作 PLOG 爆款。

### O-XHS-005｜可打印清单比生活方式封面更适合高意图搜索

- 查询 / 入口：`租房合同 避坑`。
- 笔记：`真的不要在现场直接签租房合同了！`；作者 `王律爱普法`；note ID `69f877b20000000038022a48`。
- 可见表现：约 1.3 万赞、1.7 万收藏、219 评论；收藏高于点赞。
- 主页对照：28 条近期、非置顶可见帖子互动中位数约 62，表面异常倍数约 209.68。
- 视觉机制：白/浅灰文档底，巨大的黑色衬线标题，密集编号清单，小面积蓝/橙/emoji 做分节；接近可保存、可打印的文档。
- 可迁移假设：高意图搜索/保存任务可以牺牲“生活方式感”，换取扫描性、条目完整度和打印感。
- 反证/边界：目标帖较旧，基线是近期快速发布窗口，内容年龄不可比；该倍数只能标高表现候选。清单外观也不能代替法律事实审核。
- 当前资格：`candidate-high`，不能晋级。

### O-XHS-006｜搜索教程的页面工作是“定位下一步”

- 查询 / 入口：`iPhone设置 教程`。
- 笔记：`16个iPhone17到手一定要会做的设置！`；作者 `落樱YING`；note ID `690c75c90000000004010896`。
- 可见表现：约 2 万赞、1.6 万收藏、163 评论。
- 主页对照：28 条近期非置顶可见帖子互动中位数约 735，表面异常倍数约 27.21。
- 封面：浅灰/白底，`iPhone 17` 与数量型利益点，单一手机背面渲染；信息少于内页。
- 内页：浅蓝底；顶部是粗体“序号 + 设置路径”；下一层用红色强调解决的问题；主体为两张原生系统截图，红框只圈当前操作位置；偶尔在底部补充一个结果说明。
- 可迁移假设：截图教程不追求装饰，而追求每页只有一条路径、一个被圈位置和一个可验证结果。
- 反证/边界：搜索页同时存在实拍视频、图标卡、红框截图等多种载体；红框截图本身并不稳定高表现。目标是 2025-12，主页基线主要来自 2026，时间窗不可比。
- 当前资格：`candidate-high`，不能晋级。

### O-XHS-007｜口红图文让皮肤与质地占满画面

- 查询 / 入口：`黄皮 口红试色 实拍`，切换“图文”。
- 可见样本：顶部图文约 1449、553、1001、274；同一试色账号在结果中还有约 534—1203 的多篇相似内容。
- 视觉机制：嘴唇/下半脸占画面主要区域；比较帖直接使用 2×4 嘴唇实拍网格；单品帖把唇部、膏体和极少量色块放在同一帧；部分封面完全不放标题。
- 可迁移假设：质感判断任务需要真实皮肤、原相机差异与高占比局部，不应使用 AI 生成“完美皮肤”或把产品塞进抽象 PPT 卡片。
- 反证/边界：搜索首屏并未出现压倒性高异常；类似实拍从个位数到千级都有。该机制当前只能证明“任务匹配”，不能证明“最容易获得流量”。
- 当前资格：`candidate/ordinary-control`。

### O-XHS-008｜改造爆文的资产是“变化证据 + 成本”，不是奶油风

- 查询 / 入口：`出租屋 改造 前后 实拍`，切换“图文”。
- 笔记：`慢慢把房间变成自己的样子｜改造花费大公开`；作者 `今天也很好`；note ID `6941855d000000001b02280c`；发布时间 2025-12-17。
- 可见表现：6188 赞、1839 收藏、373 评论。
- 同账号基线：主页 30 条可见帖子；排除目标和 1 条缺失计数后，28 条中位数 27.5，目标约 225.02 倍。相邻时期帖子多在 5—117，最高普通对照 664。
- 封面：直接放“难看”的原始出租屋，只有一个白底黑字标签“左滑看改造”和橙色箭头；没有先把结果美化成品牌海报。
- 第 2 页：仍是原始空间的另一视角，用一句“原始家具 / 采光”评价推动观察；10 页序列延迟展示变化。
- 正文：给出具体家具/装饰费用，最终总花费 547 元；评论集中追问低价来源和自己的房间如何改，表明内容承担证明与实用参考两种工作。
- 可迁移假设：变化类内容可用不完美的 before 作为缺口，随后给真实过程/结果，并用具体成本降低模糊感。
- 关键反证：该账号长期复用相近标题和房间实拍，绝大多数表现低。可迁移的是“可验证变化 + 具体代价 + 延迟揭晓”的叙事机制，不是出租屋、奶油风、白字或箭头。
- 限制：主页窗口跨 2025-12 到 2026-07，内容年龄并非完全一致；互动仍可能受搜索、热点或外部分发影响。
- 当前资格：强 `candidate-high`；要升级仍需另一个账号、独立内容 cluster、普通/低反例和可重算查询快照。

## 目前可以写进一次性 prompt 的内容

这些是 `curated_bootstrap/candidate_only` 指令，不是已验证的流量公式：

1. 先写 `primary_job`，再决定载体；关系阅读、搜索解答、比较决策、变化证明不能共用一套视觉骨架。
2. 真实素材应承担信息工作：聊天气泡承载推进、系统截图承载操作、皮肤特写承载质地、before/after 承载变化。
3. 每页先规定注意力路径和唯一问题，再规定字体/颜色；装饰只能服务定位、比较或节奏。
4. “反 PPT”是防止无证据的企业 KV，不是否定网格、清单和正式编辑；决策、教程、法律清单需要秩序。
5. 不完美画面可以是证据。before、原相机、生活杂物和系统界面不应被自动磨皮或替换成 AI 质感图。
6. 同载体内必须有概念差异的双原型，例如变化稿可测“before 缺口”与“after 结果”，不能只换颜色。
7. 禁止把便签、手写、荧光笔、贴纸或圆角卡片设为默认小红书味；只有当它们执行明确的信息任务时才使用。
