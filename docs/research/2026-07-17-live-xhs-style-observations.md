# 小红书站内风格观察：高表现、普通对照与适用边界

快照日期：2026-07-17—2026-07-18

采集界面：已登录的小红书 Web 搜索页、笔记详情页与作者主页。
用途：为一次性审美提示词和后续风格库提供当前站内线索。本文只记录可观察关联，不声称平台因果，也不把第三方图片或长段原文提交到仓库。

## 口径

- 搜索排序、高赞绝对值和首页位置都不能单独定义“爆款”。
- 优先比较同一作者主页中可见、非置顶、同指标的帖子；目标帖从基线成员中排除。
- `高表现候选` 表示当前快照中相对异常，不等于已经达到 Skill 的 `supported/reusable` 门槛。
- 万级数字为界面显示的近似值，不伪装成精确计数。
- 内容年龄、置顶、付费、热点、账号增长阶段或主页只展示部分帖子时，必须保留限制。

## 逐条观察

### O-XHS-001｜聊天界面直接承担故事

- 查询 / 入口：`聊天记录 故事`，Web 综合搜索。
- 笔记：`《创飞相亲对象的一天》`；作者 `今天也想创飞所有人`；note ID 仅在本轮界面记到前缀 `6a521523`，再次入正式 SQLite 前必须补全身份。
- 可见表现：约 1.7 万赞、161 收藏、492 评论。
- 主页对照：30 条可见帖子互动中位数约 2756.5，目标约 6.17 倍；同一标题/结构的其他帖子从 57 到 5.2 万不等。
- 视觉机制：灰色聊天背景，大面积系统原生白/绿气泡和留白；没有额外卡片、贴纸或品牌 KV。
- 文字推进：开场钩子 → 拒绝/冲突 → 新信息 → punchline；界面顺序本身就是分页节奏。
- 可迁移假设：当信息天然是对话时，让真实界面成为 carrier，避免把聊天再包装进 PPT 卡片。
- 反证/边界：同一载体的表现跨度极大；不能把聊天截图本身写成流量规则，也不能编造“真人故事”或伪造私聊。
- 当前资格：`candidate`；身份未补全，不能充当静态 starter 的 `observed_association`。

- 搜索对照：同一查询首屏有约 2.4 万、1403、1103、653 等候选，也有 2—282 的大量品牌分析、酒店官号、清单和实拍；“酒店细节”题材与豪华空间本身不保证高表现。

- 文案机制：先列前调邮件、拉车门、纪念日布置、数据线收纳、按电梯等可观察细节，再把付费服务转译为关系中的情绪劳动；随后用“酒店不会做什么”完成对照，最后落到珍惜自己的投入与感谢亲友。结构是 `具体体验证据 → 反常识类比 → 关系价值重估 → 自我边界`。
- 评论证据：高赞评论继续解释“标准化温柔”和“偏爱”的差别，也有人用生病时酒店能否真正响应提出反例。评论区承担概念澄清和边界补充，而不是预设的带货问答。
- 可迁移假设：观点/关系内容不一定需要信息卡。一个真实场景素材可以只承担事实锚点，正文用具体细节和可争论的类比推进；当 primary job 是关系反思时，不要把长文硬拆成九宫格 PPT，也不要让豪华空间喧宾夺主。
- 关键反证：高表现可能来自作者已有女性/关系议题信任、标题反常识和评论传播，而不只是 Live 实拍或酒店题材。同账号大量相近随笔表现低很多；不能把“住奢华酒店”或“情绪劳动”复制成通用标题。
- 安全边界：只能使用作者本人真实经历、授权匿名素材或明确虚构；不得编造酒店服务、朋友行为或“真人投稿”。涉及具体酒店服务承诺时要区分一次体验、品牌标准与普遍事实。
- 当前资格：强 `candidate-high`；需要另一个账号的真实体验→观点类比样本和低表现反例才能晋级。

## 目前可以写进一次性 prompt 的内容

这些是 `curated_bootstrap/candidate_only` 指令，不是已验证的流量公式：

1. 先写 `primary_job`，再决定载体；关系阅读、搜索解答、比较决策、变化证明不能共用一套视觉骨架。
2. 真实素材应承担信息工作：聊天气泡承载推进、系统截图承载操作、皮肤特写承载质地、before/after 承载变化。
3. 每页先规定注意力路径和唯一问题，再规定字体/颜色；装饰只能服务定位、比较或节奏。
4. “反 PPT”是防止无证据的企业 KV，不是否定网格、清单和正式编辑；决策、教程、法律清单需要秩序。
5. 不完美画面可以是证据。before、原相机、生活杂物和系统界面不应被自动磨皮或替换成 AI 质感图。
6. 同载体内必须有概念差异的双原型，例如变化稿可测“before 缺口”与“after 结果”，不能只换颜色。
7. 禁止把便签、手写、荧光笔、贴纸或圆角卡片设为默认小红书味；只有当它们执行明确的信息任务时才使用。
8. 正式内容允许“编辑设计感”，但 prompt 必须明确封面与内页分工：封面只完成问题、范围与可信压缩承诺，内页按层级完成解释；不得把每页做成同一企业宣传卡。
9. 信息图先写可核验数字、出处和页面问题，再写视觉材质；深蓝、红编号、白底网格只能作为当前方向选择，不能作为高流量装饰词。
10. 观点型长文允许“单一真实场景锚点 + 正文推进”：封面/首图不必加大字，prompt 要明确素材只证明在场，论证必须由具体细节、类比转折和边界完成；若没有真实素材则换载体，不得用 AI 酒店图冒充体验。
11. 系列化内容可以使用固定的荒诞代理人和稳定字卡系统降低识别成本，但每个选题仍要独立通过高/低对照；不得把代理词、纯色背景或大字当成流量按钮。
12. 单图字卡的页面工作只应是让人停下并识别系列，caption 必须用具体场景兑现抽象命题；若正文只是换词复述标题，则即使模板原生也不应判为 ready。
