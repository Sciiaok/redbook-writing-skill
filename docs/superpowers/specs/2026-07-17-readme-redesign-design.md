# README 改造设计：让运营者想用，让工程师信服

## 目标

把仓库首页从“功能说明”升级为一个能完成三件事的入口：

1. 让小红书内容运营者在首屏明白：这不是又一个“爆款公式”，而是一套能把调研、选题、成稿和复盘串起来的内容操作系统。
2. 让读者看到可验证的硬证据，而不是“专业、智能、全流程”等空泛形容词。
3. 让新用户在几分钟内知道适用场景、输入什么、如何安装、如何开始第一次运行。

首要行动：安装 `redbook-writing` Skill 并用一个真实类目问题启动 `discovery`、`mechanism`、`refresh` 或 `draft` 运行。

## 受众

第一受众是小红书内容运营者、创作者和做内容增长的产品/运营人员；第二受众是希望把研究和审校流程交给 Agent 的开发者。

README 的语言顺序是：先讲运营问题和结果，再展示证据账本、Schema、验证器和测试。专业术语必须同时给出一句人话解释，避免只对工程师友好。

## 叙事主线

```text
爆款传言/账号印象
        ↓
近期样本 + 普通基线 + 高低表现 + 反例
        ↓
source / claim / account / post / topic / draft 证据链
        ↓
真实性、平台规则、商业资格六道门
        ↓
primary_job + 主指标 + 代理指标 + 单变量准实验
        ↓
可复核的选题、成稿与下一轮动作
```

README 不承诺流量、转化或平台准入；“牛逼”来自可审计的判断链，而不是夸大结果。

## 页面结构

### 1. Hero：一句话定位 + 可验证结果

推荐定位：

> 把“找爆款”变成可复核的内容实验。

副标题说明它覆盖：小红书调研、机制拆解、选题/标题/封面、成稿、评论边界、跨平台承接和商业资格审核。

首屏放真实项目规模：

- 7 份核心方法参考。
- 13 份可复制的运行/账本模板。
- 130 条来源、75 条主张、59 条查询记录的研究快照。
- 84/84 自动化测试通过；严格快照校验 0 error / 0 warning。

这些数字必须和仓库文件一致，后续变更时同步更新。

### 2. 痛点对照：运营黑话逐一纠偏

用短表格解释常见说法：

| 你听到的黑话 | Skill 怎么处理 |
| --- | --- |
| CES 权重、固定 200 流量池 | 标为 `unknown`，给作用域、来源和可验证替代 |
| 冷启动、养号、流量池 | 区分平台公开证据、生产经验和待测假设 |
| 爆款、头部账号 | 分规模、近期表现、精准受众、商业邻近四种头部 |
| 账号基线 | 近期同类内容的中位数、高低位和反例 |
| 主指标、代理指标 | 一篇只选一个 `primary_job`，互动不冒充成交 |
| 外跳、承接、归因 | 拆四种 `direction`，无用户级闭环只报 `directional` |
| SKU、offer、surface | 每个商业 CTA 锁定平台/账号/素材/目的地和审批 |

### 3. 一次运行怎么走

用 Mermaid 或等价 ASCII 图展示：问题定义 → 查询树 → notes-first 采样 → 反查账号 → 基线/反例 → 选题卡 → 成稿双审 → 复盘。

### 4. 四种模式

用四张短卡片说明 `mechanism`、`discovery`、`refresh`、`draft` 的输入、输出和停止条件，避免用户把“机制研究”和“写稿”混成一次无证据创作。

### 5. 真实可用示例

至少提供三个示例入口：

- “CES 是不是 1/1/4/4/8？”如何得到未知状态、来源边界和漏斗替代。
- “研究 gay 成人亲密关系头部账号”在无法访问时如何安全停止，不伪造榜单、选题和成稿。
- “从多个平台导流卖成人玩具”如何拆 direction、六道门和 SKU 资格，不生成竞品截流或未批准 CTA。

### 6. 立即开始

保留最短安装命令，再给一个可复制的任务模板：类目、目标受众、具体处境、primary job、可用素材、商业目标和禁区。用户不需要先读完整方法论。

### 7. 可信边界与质量证明

把“不会做什么”写得像产品能力，而不是免责声明：不伪真人、不假测评、不伪造数据、不做竞品截流、不绕过登录/验证码、不把未知写成事实。

链接到测试、严格验证器、证据快照、研究方法和成稿质量规范。

## 文案规则

- 语气：专业、锋利、可执行；允许使用“小红书黑话”，但每个黑话都给出精确定义。
- 证据：只使用仓库真实数字和真实文件，不创造用户量、成功率、安装量或效果承诺。
- 结构：短段落、表格、代码块和流程图优先；避免长篇形容词。
- CTA：指向安装和第一次运行，不写“马上爆”“算法最喜欢”。
- 安全：成人亲密关系、身份、商业资格等边界继续保持现有 Skill 的严谨程度。

## 验收标准

- 首屏 10 秒内能回答“它是什么、解决什么、为什么可信、我怎么开始”。
- 至少 8 个专业术语被准确解释，且没有把经验写成平台事实。
- 至少 3 个具体场景能让读者看到输入与输出。
- 关键硬指标均可在仓库中追溯。
- README 内链接全部存在；现有测试、严格验证和 Skill 校验继续通过。
- 改造只影响 README 和本设计说明，不改变 Skill bundle 的行为或安全门。
