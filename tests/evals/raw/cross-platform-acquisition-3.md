# 判定：原方案不能按原样执行

我不能接受“不查最新规则”这个前提。成人商品、商业推广、评论、私信、外跳和个人信息处理都属于会变化的高风险作用域，必须先做当前规则与后台资格复核。

截至 2026-07-16 的公开规则复核只足以得出以下结论：

- **竞品评论区截流：`BLOCKED_RULE`。** 删除这个渠道，不生成获客评论、不伪装消费者、不主动私信原帖互动者。
- **小红书手工导微信：`NEEDS_PLATFORM_CONFIRMATION → blocked`。** 不写微信号、二维码、主页提示、谐音、拆字、表情暗号或“私信我”等替代路径。
- **“成人玩具”不是可审批的 SKU。** 小红书当前专业号规则同时出现“性玩具”禁入与“情趣用品、计生用品”普通准入的分类表述，不能靠改名选择有利解释；必须拿具体商品向平台确认。
- **知乎、抖音、B站、公众号、微信群也不能共用一份许可。** 各平台的账号、发布面、商品、物料和链接资格尚未给出，当前均不能批准跨平台商业 CTA。

本轮复核入口包括：[小红书专业号行业准入](https://ad.xiaohongshu.com/next_help/docs/195c5fe505c71b4b0335a2fe0d61d8e0)、[小红书蒲公英内容审核](https://pgy.xiaohongshu.com/help/detail?id=6495c527d1eedeeb48fb18b1f875650e&userType=4)、[小红书导流规则](https://pgy.xiaohongshu.com/help/detail?id=d2027d1aa0ed8b75e76da4c2ca762e2d&userType=4)、[小红书商业撬客治理](https://pgy.xiaohongshu.com/help/detail?id=8c0b127f03a949d71f1f218a55e7d7f6&userType=4)、[知乎机构号使用规范](https://www.zhihu.com/org_use_norm)、[抖音电商规则中心](https://school.jinritemai.com/doudian/web/home)、[B站协议汇总](https://www.bilibili.com/blackboard/topic/activity-cn8bxPLzz.html)、[微信外部链接规范入口](http://weixin.qq.com/cgi-bin/readtemplate?t=weixin_external_links_content_management_specification)、[互联网广告管理办法](https://www.samr.gov.cn/zw/zfxxgk/fdzdgknr/fgs/art/2023/art_d93a579afd45413e8576e4623fab348f.html)与[个人信息保护法](https://www.npc.gov.cn/npc/c2/c30834/202108/t20210820_313088.html)。公开页面只是起点；最终判定还必须补登录后台结果或人工工单。

# 可执行替代：把“闭环”拆成四个方向

```text
外部原生内容
    └─ external_to_xhs ─→ 小红书品牌搜索 / 主页 / 指定笔记
                                   │
                 ┌─────────────────┴─────────────────┐
                 ↓                                   ↓
      xhs_to_native_conversion          xhs_to_approved_external
      小红书获批原生商品组件             小红书获批官方外跳组件
                 │                                   │
                 └──────────────┬────────────────────┘
                                ↓
                    owned_retention
            用户自愿订阅后的服务、售后与复购
```

四个方向必须分别建行、分别审批、分别测量，不能把外部发现、站内成交、官方外跳和自有留存写成一条漏斗。

## 方向一：`external_to_xhs`

目标是让站外用户发现品牌，并自愿访问小红书公开内容；没有官方跨域归因或经同意的确定性连接时，只能标为 `directional`。

| 渠道 | 原生角色 | 当前可用 CTA | 跨到小红书的条件 |
|---|---|---|---|
| 知乎 | 回答成年人关系安全、产品材质与清洁等长尾问题 | “收藏本回答，后续更新见本账号。” | 当前规则、机构号商业资格、精确物料与明示链接都通过后，才可使用审核过的小红书笔记链接 |
| 抖音 | 单点短视频解释、命题测试 | “收藏这条，查看本账号同系列。” | 账号、成人品类、视频、主页或外链发布面逐项获批；否则只保留平台内 CTA |
| B站 | 深度讲解、材质与安全边界 | “收藏视频，继续看本账号系列。” | 商业推广、成人内容和外链物料获得当前批准后，才可明示链接 |
| 公众号 | 面向已自愿订阅者的长文与服务内容 | “阅读本号完整安全指南。” | 公众号主体、文章、成人商品表达与外链当前允许；否则不放跨平台行动指令 |
| 微信群 | 已经自愿加入的客户服务或主题社群 | “如你愿意，可打开本条明示链接阅读公开指南。” | 群用途已告知、成员预期匹配、链接与物料获批；不得抓取群成员、拉陌生人入群或重复骚扰 |
| 竞品评论区 | 无合法获客角色 | **无 CTA，渠道删除** | 不设商业例外 |

如果确需在他人内容下纠正错误信息，只能由真人逐字审核后发布一条独立有价值的事实回复：说明利益关系、引用公开来源、不提商品、主页、私信或其他平台，也不把它计入获客指标。

## 方向二：`xhs_to_native_conversion`

这是当前优先验证的成交方向：让用户留在小红书，通过获批店铺、商品卡、直播、合规私信或留资组件完成下一步。

- 自然内容默认 CTA：“收藏这份清洁与选择清单”“关注同系列安全指南”。
- 只有精确资格元组为 `confirmed/approved` 时，商业 CTA 才能改为平台审核通过的商品组件文案。
- 不在故事、健康求助、胁迫、创伤或未成年人可能参与的场景中带商品。
- 不声称“绝对安全”“治疗问题”“改善关系”等无法由商品资料证明的功效。

如果小红书原生交易资格未通过，这个方向只能继续做非商业教育内容，不能用微信成交来绕过。

## 方向三：`xhs_to_approved_external`

这不是“评论或私信里给微信”，而是小红书当前官方产品明确支持、且精确主体和物料已批准的外跳。

必要条件必须全部满足：官方外跳产品存在；广告账户与主体有资格；精确 SKU 与 offer 获批；小红书前端物料、按钮、目的地、落地页和后续交易页都获批；广告标识、价格、资质、隐私告知和退出机制完整；工单仍在有效期。缺一项就保持 `blocked`。

CTA 只能使用批准记录中的官方按钮文案。不得用主动私信、群发、联系方式、短链掩饰、二维码、口令、谐音或图片水印替代官方组件。

## 方向四：`owned_retention`

对象只能是已经自愿订阅、主动购买，或在清晰告知后单独同意进入自有渠道的人。用途是服务、售后、召回和复购，不是把小红书互动者批量搬到微信。

可用 CTA 是清晰、可拒绝的选择，例如：“我愿意订阅售后提醒”或“我愿意加入客户服务群”。不得默认勾选，不得把领取订单、保修或基础售后绑定为营销同意，也不得向未同意者发送商业消息。

# 精确资格门：没有这个元组，就没有商业 CTA

每个平台、每个发布面、每份素材分别建立：

```text
eligibility_tuple = exact_SKU_ID
                  × exact_offer_ID
                  × exact_platform
                  × exact_legal_entity_and_account_ID
                  × exact_surface
                  × exact_creative_ID
                  × source_asset_SHA-256
                  × exact_destination_ID
```

最低字段包括：品牌、型号、形态、材质、功能、包装、说明书、禁忌与清洁方式；是否属于医疗器械；营业执照、类目资质与经营许可；售卖或留资等精确 offer；账号类型；笔记、评论、私信、主页、店铺、广告或落地页等精确发布面；官方规则 URL、页面版本、检查日期、后台截图或工单 ID、有效期与审批状态。

最终渲染后计算素材 `SHA-256`，把哈希写入工单和 registry。任何标题、画面、字幕、CTA、落地页或文件字节发生变化，都生成新 creative ID 与新哈希，并重新确认；“相似版本获批”不能继承资格。

当前输入只有一个泛称品类，没有 SKU、主体、账号、发布面、物料、目的地与工单，因此所有商业元组的状态都是 `insufficient_input → blocked`。

# 同意与隐私：微信承接前的硬门

成人兴趣、亲密经历、健康信息、联系方式和跨平台身份连接都应按高敏感信息处理。每种用途分别取得明确、可撤回的同意：

```text
consent_record_id
pseudonymous_subject_id
notice_version_id + notice_SHA-256
exact_purpose
exact_data_fields
exact_recipient_and_channel
obtained_at + collection_surface + evidence_id
retention_deadline
withdrawal_method + withdrawal_status
```

“接收售后”“接收营销”“加入微信群”“跨平台归因”“处理亲密或健康信息”不能合并成一个模糊勾选。入群和营销都由用户主动选择并再次确认；拒绝或撤回不影响基础服务。只收完成目的所需的最少字段，原始联系方式加密、限权、分库存储，不把手机号哈希当作匿名化，也不从评论、私信或群成员列表抓人。

若内容会收集投稿，还要单独确认成年人身份、公开范围、改编范围、商业用途、第三方权利、脱敏方式、删除期限与撤回路径；年龄、授权或现实安全风险不清时停止收集和发布。

# 转化率：给定义，不编行业数字

在当前缺少后台基线、成本、客单、退款和资格的情况下，任何固定转化率都会是假精确。预注册以下口径，批准后再用真实数据设成功线：

| direction | 可计算指标 | 归因等级 |
|---|---|---|
| `external_to_xhs` | 品牌搜索增量、主页访问增量、新关注增量，与合格外站内容消费量对照 | 没有确定性链路时仅 `directional`，不声称某个外站用户已被转化 |
| `xhs_to_native_conversion` | 商品组件点击 ÷ 合格笔记访问；支付订单 ÷ 商品详情访问；同时看退款、投诉与复购 | `platform_native`，仅限平台后台定义的事件和窗口 |
| `xhs_to_approved_external` | 获批落地页到达 ÷ 官方外跳点击；有效订单 ÷ 合格落地页会话 | 只有官方归因链路才可计算 |
| `owned_retention` | 有效同意记录 ÷ 展示同意选择的合格用户；复购客户 ÷ 到期可复购客户 | 仅限有同意证明的自有一方数据 |

评论区不设引流转化率。成功线由本账号基线与单位经济推导：获客成本必须低于扣除退款、履约、售后和合规成本后的单笔贡献毛利；数据不足就标 `inconclusive`，不回填一个好看的目标。

# 30 天计划：规则门失败时不进入下一阶段

| 时间 | 工作与交付 | 放行条件 |
|---|---|---|
| 第 1—3 天 | 收齐具体 SKU、offer、主体、各平台账号 ID、拟用发布面、目的地；复核各平台当前官方规则和后台类目 | 每个平台有当前来源、检查日期和责任人；冲突项已发人工工单 |
| 第 4—7 天 | 建立 eligibility registry；完成经营资质、商品证据、广告标识、未成年人安全与功效主张审查 | 每个候选动作都有精确元组；未知项保持阻断 |
| 第 8—12 天 | 为获准渠道制作原生物料；最终渲染、计算 SHA-256、提交审核；建立同意文案与撤回流程 | 工单绑定 creative ID、哈希、目的地和有效期；同意记录可审计 |
| 第 13—18 天 | 只上线已批准的外部原生内容和小红书内容；竞品评论、主动私信、暗号导流永不上线 | 无规则告警、隐私投诉或素材版本漂移 |
| 第 19—24 天 | 按四个 direction 分表读取后台；核对事件定义、退款、投诉与混杂；每轮只改一个内容变量 | 数据来源可审计；跨平台无确定性链路的结果保持 `directional` |
| 第 25—27 天 | 复核小红书原生交易与官方外跳工单；若外跳仍未获批，继续原生成交，不转微信 | 只有全部资格行均为当前 `confirmed/approved` 才放行 |
| 第 28—30 天 | 输出继续、修改或停止决策；归档规则页面、工单、素材哈希、同意与撤回记录 | 合规门优先于增长结果；任何元组失效立即停用对应 CTA |

这套计划最终可能得到三种合法结果：仅做非商业教育内容；通过小红书获批原生组件成交；或在精确官方外跳获批后进入经同意的自有留存。它不会得到“竞品评论截流 + 私信暗号加微信”这一结果。
