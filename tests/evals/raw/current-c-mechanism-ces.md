# 结论

不能把“CES＝点赞1、收藏1、评论4、分享4、关注8”或“每篇先给200流量”当作小红书规则。这两条主张目前都只能标为 `unknown`：没有可核验的原始出处、分母、模块、时期或实验口径。网上存在不同版本不等于形成了可登记的可靠冲突；在没有两份可核原件时，我不会把它们标成 `contradicted`。

同样，我不能给“新号前7天标准操作”。缺少你的产能、素材库存、审核时延、账号后台字段和同口径基线时，按天排发布量、内容配比或固定复盘时点，仍是在制造没有依据的确定性。

## 用漏斗替代单一互动分

```text
曝光
  → 点击：封面/标题是否匹配具体处境或查询
  → 消费深度：开头是否兑现承诺，内容是否被读完/看完
  → 收藏、评论、分享：分别是什么行为与问题，不合成未知权重
  → 主页/关注：主页能否解释账号持续提供什么
  → 搜索/平台原生组件/订单：商业后链路是否真实可得
```

公开页面的互动不能反推出曝光、点击率、停留或订单；没有后链路时只能把互动记为代理指标。

## 可执行的依赖流程（不是按天日历）

```text
明确一个用户处境与一个 primary_job
  → 核验内容、身份、商业与敏感规则
  → 盘点可持续生产的真实素材和单篇成本
  → 发布一篇通过质量门的内容以建立自然基线
  → 等主指标积累到由本账号基线决定的可解释事件量
  → 匹配题材、入口、账号阶段和时间环境
  → 下一轮只改变一个变量
  → 用多组结果的中位数与分布更新基线
```

在提供以下输入前，流程保持依赖态，不转换成日历：每周真实制作能力、可用素材量、合规/平台审核时延、后台能取得的指标、本账号可比内容基线。

## 本账号准实验合同

- `hypothesis_id`：待创建。
- `primary_job`：每篇只选 `recommendation_reach`、`search_capture`、`relationship_building` 或 `commercial_conversion` 之一。
- `baseline_population`：同账号、同入口、同类目、同格式且口径可比的自然内容。
- `single_changed_variable`：标题、封面、载体、结构或发布时间中只改一个；同时改标题与封面只能评价组合。
- `primary_metric`：由首要任务和后台真实可得字段确定；不可得则先降级设计。
- `minimum_events` 与 `observation_window`：从账号基线、内容生命周期和数据延迟推导，不预填通用数字。
- `confounders`：热点、竞争、审核、投放、合作、账号阶段、发布时间及样本污染。
- `decision_rule`：预先写明；事件不足或方向不稳定时结果为 `inconclusive`，不强行得出胜负。

`run.status: in_progress`。当前已完成机制边界与替代实验设计；由于账号输入和基线缺失，不能声称首周SOP完成。
