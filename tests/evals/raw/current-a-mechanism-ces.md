# 判定

两种说法都不能当作小红书事实或新号 SOP：

- `CES=点赞×1+收藏×1+评论×4+分享×4+关注×8`：拒绝使用。
- “每篇先给 200 流量”：拒绝使用。

## IDProxy 的正确作用域

`IDProxy` 是 2026-03 公开、仍属 **under-review research（审阅中的预印本研究）** 的生产团队材料。它报告的是 2025 年 **Explore Feed Content Feed / Display Ads 的 CTR 排序实验**：新内容缺少历史行为时，系统可用文字和图像形成多模态代理表示，并观察点击、阅读时长及互动等结果。它不能被泛化为小红书全站算法，也不能证明新号养号、固定首轮曝光或任何 CES 权重。

## 用就绪链代替“前七天标准操作”

当前缺少账号目标、可用素材、单篇制作成本、审核时延、后台可见字段和自然基线，不能把依赖关系伪装成日历。可执行顺序是：

```text
明确一个用户承诺与一个 primary_job
→ 确认素材真实、规则适用且内容能独立兑现承诺
→ 建立本账号同口径自然基线
→ 生产一篇通过质量门的内容
→ 等主指标达到可解释事件量
→ 与匹配基线比较，并检查审核、热点、投放和竞争等混杂
→ 下一轮只改变一个主要变量
→ 更新基线；事件不足则记为 inconclusive
```

“可解释事件量”必须从本账号的事件可得性、基线波动和决策成本推导；在事件不足时，不以单篇高低表现下算法结论，也不追加无依据的发布频率。

## 新号自然内容准实验合同

```yaml
hypothesis: 某一个表达变量是否改善指定 primary_job
baseline_population: 本账号同主题、同入口、同阶段且可比的自然内容
matched_context: 受众处境、题材、内容深度、商业力度与时间环境尽量匹配
single_changed_variable: 标题角度、封面层级、开头、载体或 CTA 中只选一个
primary_metric: 与 primary_job 对应且后台确实可取的一个指标
metric_definition: 明确分母、事件定义、数据源与缺失处理
diagnostic_metrics: 曝光、点击、消费深度、互动、主页或后链路按可得性分层诊断
confounders: 审核、热点、竞争、投放、合作、发布时间与账号阶段
observation_window: 由内容生命周期、数据到达和基线分布推导
minimum_events: 由本账号基线推导的可解释事件量，不使用通用阈值
decision_rule: 与匹配基线的分布比较；方向不稳或事件不足即 inconclusive
next_iteration: 只改一个变量并重新登记
```

这套流程学习的是本账号在特定条件下的表达差异，不是随机 A/B，也不能反推出全站算法权重。
