# 结论

不能把“CES＝点赞×1＋收藏×1＋评论×4＋分享×4＋关注×8”或“每篇先给 200 曝光”当作小红书事实；两条主张目前都应标为 `unknown`。这里的 `unknown` 是“公开原始证据不足”，不是“已经证明不存在”，也不是 `contradicted`。同样，没有可核原件支持新号存在一套按前 7 天执行的统一 SOP，因此我不会把缺失的产能、素材、审核时延和账号基线补成按天日历。

以下判断使用 2026-07-16 已复核的证据快照；本次没有联网复核页面的当前可用性。

### 原始来源与边界

- `OFF-001`：国家网信办算法备案 PDF，[原件](https://beian.cac.gov.cn/api/static/fileUpload/principalOrithm/additional/user_c015445c-80ac-45f7-94d7-3871e961b1fe_d4425f3b-7f35-45af-b8d4-badd4424d6d5.pdf)。作用域是备案所述个性化笔记推荐；能支持“内容与互动参与推荐”，不能支持 CES、固定曝光池或发布时间阈值。
- `TECH-001`：王树森《小红书的推荐系统》，[原始课件](https://raw.githubusercontent.com/wangshusen/RecommenderSystem/main/Slides/01_Basics_01.pdf)。作用域是 2022 年公开技术课件；能支持分层漏斗和系统实验阶段，不能代表 2026 年完整系统或普通创作者的统一阈值。
- `TECH-003`：IDProxy，[原始预印本](https://arxiv.org/html/2603.01590v1)。作用域是 2025 年 Explore Feed Content Feed / Display Ads 的生产实验、2026 年发布的预印本；能支持 item cold-start 与多模态代理表征进入 CTR 排序，不能支持“每篇固定先给几百曝光”。

## 用漏斗替代一个加权总分

```text
曝光
  ↓ 封面、标题与当前兴趣或查询是否匹配
点击
  ↓ 首屏是否兑现承诺
消费深度
  ↓ 内容是否值得保存、讨论或分享
赞 / 藏 / 评 / 分享（分别记录，不加权合并）
  ↓ 主页能否解释账号持续提供什么
主页访问 / 关注
  ↓ 商品证据、资格与承接路径是否成立
品牌词搜索 / 合规组件或私信 / 订单 / 退款与投诉
```

每篇先指定一个 `primary_job`，再选与它同分母的主指标。例如，任务是推荐触达且后台能取得曝光和点击时，主指标可设为 `CTR = 点击 / 曝光`；消费深度、互动、主页访问和关注只作分层诊断。若目标是成交，订单、退款与投诉才是后链路结果，点赞不能代替它们。前台只见合并互动时，应原样记为合并互动，不能反推曝光、CTR、停留或订单。

## 新号的可执行依赖链（不是按天排期）

```text
明确一个用户的具体处境与账号持续承诺
  → 盘点真实产能、可用素材、审核时延和后台可见字段
  → 通过规则、真实性与版权门
  → 选择单一 primary_job，并预先定义指标分母
  → 发布符合相同质量门的校准内容，建立本账号同口径自然基线
  → 基线不确定性已小到足以识别业务上有意义的差异
  → 预注册一个单变量准实验
  → 等两组都达到推导出的最小有效事件量且成熟度可比
  → 判为 win / loss / inconclusive，记录混杂并更新基线
```

生产节奏只能由“可用工时 ÷ 单篇合格制作与审核成本”反推；观测结束点只能由指标的实际累积速度、内容生命周期和最小事件量反推。若这些输入仍为空，就停在相应依赖门，不把它改写成每天发什么。正常浏览、发布真实内容即可；养号、互赞互粉、互动群和组织互动既没有这里的一手机制依据，也会污染自然基线。

## 单变量准实验预注册

先把数字输入补齐，而不是先给结论：

- `p0`：从本账号“同一入口、相近题材、相近账号阶段、相同指标定义”的校准样本估计基线 CTR；不能拿平台传言或别人的均值代替。
- `delta`：业务愿意为之改变生产决策的最小 CTR 差异，由制作成本与预期价值确定。
- `alpha`、`beta`：在实验开始前约定的误判容忍度；本次不擅自填固定值。
- 校准停止条件：基线区间已经窄到可以区分 `delta`；未达到时继续校准，不能宣称实验准备完成。

以“封面主文案表达”作为唯一改变变量：

```yaml
hypothesis_id: COVER-PROBLEM-QUESTION
primary_job: recommendation_reach
baseline_population: 本账号同入口、相近题材与相近账号阶段的校准内容
matched_context: 题材需求强度、标题承诺、视觉底图、正文结构、发布时间环境尽量匹配
single_changed_variable: 封面主文案使用“具体问题句”或“情绪判断句”
primary_metric: CTR
metric_definition: 后台点击数 / 后台曝光数；两组使用同一字段口径
diagnostic_metrics: 消费深度、分别记录的互动、主页访问与关注
confounders: 题材热度、竞争、审核、投放、合作、素材质量、发布时间环境、账号阶段
observation_window: 不按固定小时；从发布起累积到两组达到 n_min 且内容成熟度可比
decision_rule: 差异达到或超过 delta 且不确定区间排除无差异则 win；反向则 loss；事件不足、方向不稳或混杂无法排除则 inconclusive
```

令 `p1 = p0 + delta`，`p_bar = (p0 + p1) / 2`。若采用双侧两比例近似，单组最小有效曝光事件量在预注册时按下式计算：

```text
n_min = ceil(
  [ z_(1-alpha/2) * sqrt(2*p_bar*(1-p_bar))
    + z_(1-beta) * sqrt(p0*(1-p0) + p1*(1-p1)) ]^2
  / delta^2
)
```

这个 `n_min` 是由本账号基线、最小有意义差异和预注册误判容忍度推导的曝光事件下限，不是固定篇数，也不把自然发布伪装成随机 A/B。两组都未达到 `n_min` 时只能继续积累或记为 `inconclusive`；若后台拿不到曝光与点击，就不能事后换分母“救结果”，而应在实验前改选一个真实可得的主指标并重新推导最小事件量。
