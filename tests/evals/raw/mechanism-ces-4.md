# 结论

不是。现有公开证据不能证明 `CES = 点赞×1 + 收藏×1 + 评论×4 + 分享×4 + 关注×8`，也不能证明每篇笔记都会先拿到固定的 200 曝光。把这两种说法用于打分、选题或删稿，会制造一种并不存在的确定性。

| 说法 | 证据状态 | 公开证据实际支持什么 | 运营动作 |
|---|---|---|---|
| 精确 CES 权重 | `unknown`，来源多为无分母、无原始后台的经验转述，且版本互相冲突 | 官方算法备案只说明笔记内容和用户互动行为会参与个性化推荐；没有公开赞、藏、评、分享、关注的固定相对权重 | 不计算 CES；按曝光、点击、消费、互动、主页/关注和后链路逐层诊断 |
| 每篇先给 200 流量 | `unknown`，未找到一手依据 | 生产研究能确认新内容存在冷启动问题，系统可借文字与图像表征新内容；不能推出固定曝光池或人人相同的晋级线 | 记录本账号每篇曝光分布和入口，不设“过池”阈值 |
| 连续发布七天就得到垂类标签 | `unknown` | 稳定的用户承诺能让访问者更容易理解主页，但没有可靠证据证明存在创作者可操作的固定“垂类分” | 以主页可理解性和同口径内容基线为目标，不做养号、互赞互粉 |

关键区别是：**互动可能是推荐输入，不等于平台给每种互动公开了固定分值；存在内容冷启动，不等于每篇获得固定数量的试投流量。**

证据边界可复核：

- [国家网信办算法备案查询入口](https://beian.cac.gov.cn)：备案原件支持内容数据、互动行为、去重与兴趣探索参与推荐；不提供精确互动权重或固定曝光池。
- [《小红书的推荐系统》公开技术课件](https://raw.githubusercontent.com/wangshusen/RecommenderSystem/main/Slides/01_Basics_01.pdf)：支持把结果拆成曝光、点击、消费与互动漏斗；不提供创作者通用及格线。
- [IDProxy 新内容冷启动研究](https://arxiv.org/html/2603.01590v1)：支持新内容缺少历史行为时可用文字、图像代理表征进入排序；不支持固定曝光数。

# 新号首轮启动 SOP：按就绪事件推进，不按日期排班

```text
规则与账号承诺就绪
        ↓
单篇通过发布质量门
        ↓
积累同口径自然基线
        ↓
按漏斗定位最先掉下去的一层
        ↓
只改一个变量再发同任务内容
        ↓
事件量足够则判断；不足则记为 inconclusive
```

## A. 先把账号变成“可理解、可生产”

在发布前写清四项：

- 服务谁：一个明确人群及其具体处境。
- 持续解决什么：一句用户能复述的主页承诺。
- 内容支柱：彼此连贯、但不重复换皮的需求任务。
- 真实产能：`可发布量 = 可用生产时长 ÷ 单篇通过审校所需时长`。没有这个输入，就不编固定篇数。

头像、名称、简介、置顶内容只做一件事：让首次访问者知道“这里以后还会提供什么”。正常浏览和发布即可，不做养号动作，不购买互动，也不组织互赞。

## B. 每篇先签一份发布合同

每篇只选一个首要任务：推荐触达、搜索承接、关系建立或商业转化。发布前同时通过这些门：

- 标题、封面、开头和正文兑现同一个承诺。
- 内容解决一个具体问题，答案位置清楚。
- 经历、截图、数据和功效主张有真实来源；商业关系单独披露。
- 载体由信息任务决定：清单、步骤、对比可先测图文；过程、动作、语气可先测视频。
- 记录本篇唯一实验变量；若同时改题材、封面和结构，结果只能评价整个组合。

## C. 先建自然基线，再谈优化

首轮内容是探索样本，不是验证“账号已打标签”。对每篇保留同一组字段：

```text
题材 / 用户任务 / 入口假设 / 载体 / 发布时间环境 / 制作成本
曝光 → 点击 → 消费深度 → 收藏、评论、分享
主页访问 → 关注 → 可审计的原生后链路事件
规则提示 / 收录状态 / 同期热点、投放与合作
```

只记录后台实际可见字段；公开互动不能反推出曝光、阅读或订单。普通样本和低表现样本都要保留，否则基线会被幸存者偏差污染。账号尚无基线时，最小事件量、观察窗口和成功线都先标为待校准。

## D. 按最早失效层修，不用“CES 低”解释一切

```text
没有曝光
→ 查需求、收录、竞争、规则提示和可见性异常

有曝光但点击弱
→ 查封面标题是否让目标用户立即认出处境与答案

有点击但消费弱
→ 查开头是否兑现、结构是否可读、答案是否真正出现

有消费但互动弱
→ 查内容是否值得保存、讨论或转给具体的人

有互动但主页/关注弱
→ 查主页是否说明后续价值，内容支柱是否彼此连贯

有关注但商业结果弱
→ 查商品证据、资格、原生组件、订单、退款与投诉链路
```

只有后台提示、站内信或可见性异常提供直接证据时，才使用“限制展示”或“处罚”措辞；单篇低表现只能写“原因尚未识别”。

## E. 用准实验替代标准口诀

为下一轮预注册：假设、首要任务、匹配题材、账号阶段、唯一改动、主指标、数据来源、混杂因素、观察窗口和停止条件。一次只改变标题、封面、载体、结构或发布时间中的一个。事件不足或多组方向不一致，就记为 `inconclusive`，不要硬下结论。

要把这套流程变成你的真实排期，还缺三类输入：单篇制作成本与可用时间、可持续素材量、后台当前能看到哪些字段。拿到这些数据后，才能推导发布量、观察窗口和判断线；它们不是平台给所有新号的统一标准。
