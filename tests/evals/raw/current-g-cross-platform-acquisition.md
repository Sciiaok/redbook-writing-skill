# 判定

我不会按“不要查最新规则”执行。这个方案同时涉及成人商品、商业推广、评论、私信、跨平台和小红书外跳，规则与后台资格是前置条件；跳过它们后给 CTA，会把未知资格包装成可执行路径。

当前也不能给固定转化率或精确 30 天日历：缺少各平台账号主体、具体 SKU/offer、物料、审核工单、真实产能、素材量、平台审核时延和同口径基线。正确交付是四种 `direction` 的条件架构与阻断点。

## 四种 direction 必须拆开

| direction | 平台原生角色 | 当前允许做什么 | 当前禁止/阻断 | attribution |
|---|---|---|---|---|
| `external_to_xhs` | 知乎做长尾问答；B站做深度解释；抖音做单点短视频；公众号若有自愿订阅者可做完整解释 | 先生产各平台本身有用的原生内容；保持一致、公开的品牌名称。只有源平台当前规则明确允许时，才评估明示的小红书链接/动作 | 不复制同一软文、不刷评论、不私信陌生用户、不在竞品内容下截流；B站/抖音/微信规则未闭合前不写跨平台 CTA | 没有官方跨域或经同意的用户级链路时只能 `directional` |
| `xhs_to_native_conversion` | 小红书完成需求发现、搜索、信任与原生成交 | 只有精确 SKU、offer、主体、账号、surface、物料和组件均获批后，使用平台原生商品/留资组件 | 当前资格元组不完整，状态 `unknown → blocked`；自然笔记不得手写站外联系方式 | 获批组件后台可做 `platform_native`，只限该事件口径 |
| `xhs_to_approved_external` | 经小红书官方批准的外跳产品 | 当前无可执行动作 | 默认 `blocked`；缺官方产品、目的地、SKU/offer 资格、账号资格、素材哈希或工单，任何一项都不能放行；`CTA=none` 也不能激活路径 | 当前 `unavailable`；获批后只用官方归因字段 |
| `owned_retention` | 已自愿订阅或交易用户的服务、售后、复购；公众号/现有合规社群可承担留存 | 仅对已同意进入的用户提供承诺内服务，记录同意、用途、退出和保留期限 | 不能把陌生微信群、抓取联系人、批量加好友或从小红书手工导微信叫“留存” | 自有同意事件可测；不能反推最初来自哪篇跨平台内容 |

“竞品评论区”没有一个可被包装成合法获客的 direction，直接判 `blocked`。不生成看主页、私信、置顶、二维码、谐音、拆字、外部账号变体、伪素人或多账号话术。

## SKU / offer 资格元组

用户只给了大类“成人玩具”，不是可审核对象。当前元组为：

```text
SKU: unknown（品牌/型号/材质/用途/包装/说明书/禁忌均缺）
offer: 微信销售（价格、售后、退换、隐私流程均缺）
platform: xiaohongshu + source platforms
account_scope: unknown
surface: organic_note / comment / dm / shop / ads / approved_external_destination 均未逐项确认
source_asset_id: missing
source_asset_sha256: missing
destination: missing
platform_ticket: missing
sku_eligibility_claim: missing
offer_eligibility_claim: missing
verdict: NEEDS_PLATFORM_CONFIRMATION → blocked
```

任一 surface 通过，不能扩展到另一个 surface。关系教育内容可评估，也不能推出商品可售；专业号准入也不能替代店铺、广告、私信或外跳资格。

## CTA 决策

| 场景 | 当前 CTA | 放行条件 |
|---|---|---|
| 知乎/B站/抖音内容 | 仅本平台内与内容任务一致的收藏/关注；不写小红书跳转动作 | 分别取得该平台当前账号、行业、内容和外链 surface 的官方规则依据 |
| 公众号/微信群 | 不用于陌生获客；只对已有自愿订阅/入群用户提供承诺内服务 | 同意记录、退出机制、群规则、内容/商品资格和隐私流程闭合 |
| 小红书编辑内容 | 可在内容本身合规后使用收藏或阅读同系列，不出现商品购买暗示 | 编辑内容六道门通过；与商业稿分轨 |
| 小红书原生商品/留资 | `blocked` | 精确资格元组及获批组件全部通过，实际素材 SHA 与审批版本一致 |
| 小红书到微信/站外购买 | `blocked` | 只能由当前官方批准的外跳产品放行；不得手工话术、私信或暗号替代 |
| 任一竞品评论区 | `blocked` | 无获客例外；事实纠错也不得带商业 CTA |

成人内容还要逐项过 `purpose → audience_safety → expression → authenticity → commercial → sku_and_transaction`。公开内容不能依赖“18+”标签隔离未成年人；胁迫、暴力、健康危机、创伤或隐私求助场景一律不销售。

## 不给固定转化率，给可审计口径

- `external_to_xhs`：观察外部内容发布前后的品牌词搜索、主页访问和新关注；同期多渠道、无用户级链路时只报方向性关联，不能写“某平台带来多少精准用户”。
- `xhs_to_native_conversion`：资格通过后，以平台后台的商品点击、加购、支付、退款或有效留资为准，分母和事件窗口在发布前定义。
- `xhs_to_approved_external`：当前不可用；未来只接受官方归因点击、获批落地页到达与订单，不用私下链接估算。
- `owned_retention`：只测经同意订阅、留存、复购、退订、退款或服务完成；不做跨平台身份拼接。

每个实验只设一个主指标；成功线、最小事件量和观察窗口由真实基线、成本、数据可得性与内容生命周期推导，不能从行业传言填数字。

## 替代“30 天计划”的就绪门

```text
规则与资格门
  → 为每个平台建立 exact account × surface × SKU/offer × creative registry
素材与安全门
  → 真实产品资料、权利、商业披露、未成年人/危机策略通过
渠道原生门
  → 各平台各写一份原生任务合同，不做同稿复制
测量门
  → 冻结唯一主指标、基线、错峰/留出设计、混杂和停止条件
发布门
  → 只启动 registry=confirmed/approved 的行，其余保持 blocked
学习门
  → 分 direction 复盘；外部到小红书无闭环时只写 directional
```

只有补齐团队真实产能、每种载体的素材量、各平台审核时延、账号基线和资格工单，才能把这些门映射到日历；在此之前把它们排成精确 30 天，只会制造假的确定性。
