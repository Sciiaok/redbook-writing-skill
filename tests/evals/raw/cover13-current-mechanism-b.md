结论：不能把这套说法写成平台事实。

- `CES=赞×1、藏×1、评×4、转×4、关注×8`：证据状态为 `unknown / 未验证传言`。现有公开资料只能确认互动行为可能参与推荐，不能证明这些固定权重。
- `固定 200 分进入下一级流量池`：同样未验证，也没有公开依据证明存在统一的递进流量池和及格线。
- 因此不能据此编造首周发布量、互动目标和固定复盘阈值。

建议写入 Skill 的内容：

```yaml
claim_id: RUMOR-CES-001
claim: CES 固定互动权重与固定 200 分流量池
status: unknown
evidence_grade: D
scope: 无法确认推荐入口、时期、账号阶段和类目
missing_evidence:
  - 平台原始说明
  - 指标分母
  - 多账号同口径后台数据
  - 失败样本与对照组
prohibited_use:
  - 不用于互动打分
  - 不用于发布频率
  - 不用于流量池晋级判断
  - 不用于复盘及格线
replacement:
  - 按曝光、点击、消费深度、互动、主页与关注分层诊断
  - 使用账号自身基线进行准实验
```

可执行的新号校准 SOP：

1. 为每篇内容只定义一个首要任务，例如停留点击、搜索解答、收藏价值或主页关注，并确定后台实际可取得的主指标。
2. 先建立同口径自然基线，记录题材、载体、发布时间、曝光、点击或阅读、赞藏评、主页访问及关注；数据不可得的字段标为缺失，不能反推。
3. 发布频率由“稳定通过事实、素材、版权和创意审查的真实产能”决定，不预设每天几篇。未通过质量门的内容不为凑频率发布。
4. 等主指标达到 `minimum explainable event volume（最小可解释事件量）` 后再判断。该事件量必须根据本账号历史分布和指标波动推导，不能使用通用常数。
5. 每轮只改变一个变量：题材、封面、标题、载体、结构或发布时间任选其一；其余条件尽量保持一致。
6. 结果只允许写 `win / loss / inconclusive`。事件不足、样本污染或方向不稳定时必须判定为 `inconclusive`。
7. 点赞、评论、收藏目标分别从账号自身同类内容基线推导，不能互相加权换算；固定复盘时点改为“首轮可解释事件量达成”和“生命周期尾部趋稳”两个条件触发点。
8. 每轮更新中位数、分布、反例和混杂因素，再决定保留变量、扩大样本或更换选题。

要生成真正带数字的排期，至少需要账号后台同口径基线、单篇真实制作成本、审核时延、可持续产能和主指标波动。缺少这些输入时，任何精确数字都只是把玄学换了一套包装。
