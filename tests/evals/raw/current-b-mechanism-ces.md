# 结论

不是。`点赞×1、收藏×1、评论×4、分享×4、关注×8` 不是可核验的小红书 CES 公式；“每篇先给 200 流量”也不是可核验的平台规则。前者不能拿来算分，后者不能拿来排新号启动计划。

### IDProxy 的边界

IDProxy 的来源状态是 `under-review research`：它是 2026-03 发布、尚未同行评审的生产研究预印本。它披露的是 **2025 年 Explore Feed 的 Content Feed 与 Display Ads CTR 排序生产实验**。在这个精确范围内，它支持：新内容缺少历史行为时存在 item cold-start，文字与图像可形成多模态代理表示接入 CTR 排序。它不支持把结论推广到小红书全站、搜索、所有推荐面或当前完整系统，更不支持固定首轮曝光、CES 权重和“新号必须养几天”。

## 新号启动：就绪依赖链，不做首周日历

```text
账号承诺与 primary job 明确
→ 当前规则、真实性与素材权利通过
→ 后台可取指标及口径确认
→ 建立同账号自然基线
→ 完成一篇通过合规与创意审查的内容
→ 主指标达到可解释事件量，或由内容生命周期推导的观察窗结束
→ 只改一个变量做账号内准实验
→ 记录混杂、更新基线，再决定保持、迭代或判定 inconclusive
```

这条链的前置输入还缺：账号面向谁、用户在什么具体时刻需要内容、账号的首要任务、可用素材与单篇制作成本、后台能读取哪些字段、审核时延、自然内容基线。没有这些输入，任何按日期写的固定发布量、内容配比、标题长度或统一复盘点都只是伪确定性。

## 账号内准实验合同

| 字段 | 本次应如何填写 |
|---|---|
| `hypothesis` | 写成可证伪表达，例如“在相同用户任务下，直接说明答案的封面是否比只制造悬念的封面带来更好的合格消费” |
| `primary_job` | 先在 `recommendation_reach`、`search_capture`、`relationship_building`、`commercial_conversion` 中选一个，并写清用户完成的单一任务 |
| `baseline_population` | 同账号、同入口、同题材任务、相近账号阶段的自然内容；不得拿大号爆文作基线 |
| `single_changed_variable` | 标题角度、封面层级、开头、载体、发布时间中只改一个；若同时改动，只能评价组合，不能拆因果 |
| `primary_metric` | 按任务选择后台真实可取的一个主指标；曝光、点击、消费深度、互动、主页与关注分层诊断，不合成 CES |
| `minimum_events` | 从账号基线、指标波动与数据可得性推导“可解释事件量”；未达到时写 `inconclusive`，不强行下结论 |
| `confounders` | 记录热点、竞争、审核、投放、合作、发布时间、账号阶段和样本污染 |
| `decision_rule` | 与匹配基线比较分布和中位数；方向稳定且事件充分才更新判断，否则继续补匹配样本或停止 |

因此，新号的标准动作不是“照公式冲权重”，而是：先让账号承诺、内容证据、规则和测量口径就绪，再用同账号准实验学习哪个表达在这个账号、这个入口、这个用户任务下更有效。
