# 结论

不是。现有可核验的一手材料不支持“赞 1、藏 1、评 4、转 4、关 8”的 CES 权重，也不支持“每篇先给 200 流量”。它们不能用来算分、预测曝光或设计新号动作。

我也不会把“先声明不是平台规则”当免责，然后另编一套首周篇数、内容配比或固定复盘时点。你还没有提供账号产能、真实素材、审核时延和后台基线，因此目前能给的是一条依赖证据的就绪链，不是按天排定的日历。

## 替代模型：先看漏斗，不看神秘总分

```text
一方 impressions（平台只有 reach 时固定用 reach）
  → 点击/打开：标题、封面与入口是否匹配
  → 消费：首屏是否兑现，是否读到答案
  → 任务结果：这篇唯一 primary_job 是否完成
  → 互动语义：在问目标问题，还是被争议/颜值带偏
  → 主页/关注：账号承诺是否清楚
  → 合规原生转化：资格、组件、订单、退款另行核验
```

没有一方 `impressions/reach` 时，流量结论就是 `unavailable`；点赞、收藏、评论、CTR、停留或成交都不能顶替曝光。

## 新号可执行的“就绪链”

- **上下文门：**先确认目标用户的具体处境、账号的一个稳定承诺、可用真实素材、单篇制作成本、审核时延和内容禁区。
- **基线门：**按同入口、同任务、同载体保存自然发布数据；事件量不足时只记 `inconclusive`，不把低流量解释成限流。
- **生产门：**每篇只选一个 `primary_job`，标题、封面、首屏和正文兑现同一个承诺；真实性、授权、商业披露先通过。
- **测量门：**预先写唯一主指标、数据源、分母、护栏、混杂和停止条件。观察窗口由账号数据可得性和内容生命周期推导，不使用通用固定时点。
- **学习门：**在相近主题、账号阶段和时间环境下，一轮只改变标题角度、封面层级、开头、载体或 CTA 中的一个变量；比较分布与中位数，结论只写 `win / loss / inconclusive`。

要把它变成真实排期，最小还缺四类输入：你每个生产周期能完成的合格内容量、已有真实素材清单、平台/团队审核时延、以及后台可取的同口径基线。补齐后，排期数字应由这些约束推导，而不是由“新号前七天”倒推。
