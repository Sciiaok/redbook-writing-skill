## 安全续跑时做什么

### 1. 先查笔记，不先猜账号

下一次具有合法可读访问条件时，先按八类查询建立候选池；下面只是 **query candidates，不是选题、标题或成稿**：

| 查询组 | 查询目的 | 示例查询方向 |
|---|---|---|
| A 类目核心 | 确认公开供给边界 | 同性伴侣关系、男男亲密关系沟通 |
| B 人群处境 | 找明确自述的具体处境 | 出柜后的伴侣沟通、异地同性伴侣 |
| C 场景任务 | 找正在解决的问题 | 关系边界怎么谈、拒绝后怎么沟通 |
| D 痛点障碍 | 找失败与阻力 | 亲密需求不一致、隐私压力 |
| E 方案动作 | 找可执行做法 | 同意边界清单、冲突复盘 |
| F 结果期待 | 找用户要完成的变化 | 更安全地表达需求、建立共同规则 |
| G 内容载体 | 比较信息任务 | 关系科普、情境演练、授权案例拆解 |
| H 反例质疑 | 主动找反证 | 没用、不适合、越聊越糟、过度概括 |

每个核心查询分别记录综合、热门、最新入口、采集时间和个性化环境；搜索位置只用于复现，绝不写成全平台排名。最多执行四轮查询树，若出现登录墙、验证码、频率限制或循环跳转，立即更新本检查点并停止：不反复重试、不绕过访问控制。

### 2. 从聚焦笔记反查账号

只有先完整读到笔记标题、正文、逐页轮播/视频、公开指标和必要评论后，才记录作者账号并进入主页。对每个重点账号采近期连续、非置顶、同类普通内容，按同一可见指标建立中位数；目标高帖必须从 baseline 排除。再保留：

```text
近期连续普通基线
+ 同账号高表现异常
+ 同任务普通/低表现对照
+ 独立反例与质疑评论
```

公开页若只显示合并互动，就原样保存，不能拆成点赞、收藏、评论。公开互动最多形成 `public_proxy_association`，不称曝光或真实流量。

### 3. 敏感身份边界

研究对象是“内容讨论了什么”，不是替作者判定性取向。不得依据头像、昵称、外貌、语气、关系照片或评论推断作者是 gay。只有作者在当前可见内容里明确自述且该字段对研究必要时，才可记录“页面明确自述”，仍需最小化保存；否则身份字段为 `unknown`。对账号的“精准受众”判断也必须来自反复议题与匿名评论问题，不能从刻板印象推导。

### 4. 进入选题与成稿的证据门

- `active` 选题：至少绑定两个不同账号的可追溯内容样本，并有反例。
- `experimental` 选题：仍至少绑定一条真实需求或内容样本。
- 零样本：只能保留上面的研究问题和 query candidates；不得输出选题、标题、封面、聊天记录或成稿。
- 封面/文风库还必须有目标帖、同账号 baseline、匹配普通/低对照和逐页 observation；单篇高互动截图不能升级成“爆款风格”。
