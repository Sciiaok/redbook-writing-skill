结论：这套 CES 权重公式目前属于 `unknown / 未验证传言`，固定 200 分进入下一级流量池同样未经证实，不能写成平台事实，也不能据此制定发布和复盘阈值。

可以写进 Skill 的版本是：

```yaml
claim_id: CES-RUMOR-001
claim: CES 精确互动权重与固定 200 分流量池
status: unknown
evidence_grade: D
missing_evidence:
  - 当前官方原始说明
  - 明确适用入口与时期
  - 指标分母
  - 可复核后台样本
skill_action:
  - 禁止用于打分、排期或预测流量
  - 按曝光、点击、消费深度、互动、主页/关注分层诊断
```

公开证据只能支持平台会综合内容和互动信号，并存在曝光、点击、阅读、互动等不同环节；不能推出赞、藏、评、转、关注的精确相对权重。

新号 SOP 应改为条件式：

```text
确认主任务与后台可取指标
→ 通过真实性、证据、封面和正文质量门
→ 按真实产能发布一篇合格内容
→ 等主指标达到最小可解释事件量
→ 定位问题所在的漏斗环节
→ 每轮只改变一个变量
→ 累积同口径样本并更新账号基线
```

发布频率由素材供给、单篇制作成本和审核能力决定；没有这些输入，不能给出每天固定篇数。

复盘时点由内容生命周期和事件积累速度决定，而不是统一时钟：

- 曝光不足：检查需求、收录、竞争和平台提示。
- 有曝光但点击弱：检查封面、标题和人群匹配。
- 点击后消费弱：检查开头兑现、信息密度和结构。
- 消费正常但收藏、评论或关注弱：检查实用价值、讨论接口和主页承诺。
- 数据量尚不足：标记 `inconclusive`，继续观察，不强行下结论。

阈值应从账号自身同类内容的中位数、波动范围和最低可解释事件量预注册推导。拿到后台基线和真实产能后，才能生成带具体数字、来源和停止条件的排期表。
