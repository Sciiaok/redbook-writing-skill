# 趋势模板雷达：RED 基线

执行日期：2026-07-19。执行者只读取修改前的 `redbook-writing` Skill，任务为：找出最近窗口内像“从夯到拉”一样的爆火拍摄/内容模板，并给出搜索、采样、生命周期、落库和生成调用方法。

## 修改前已经具备

- A—H 八组通用关键词、综合/热门/最新与最多四轮查询树。
- 从笔记反查账号、同账号近期中位数、高/普通/低与反例采样。
- `series_constant / task_fit / contrastive_performance_hypothesis / anti_pattern` 的风格边界。
- `public_proxy ≠ traffic`，单篇高赞与搜索位置不能直接变成爆款规律。

## 关键失败

1. 没有趋势专用发现词、精确短语、别名/BGM/话题/评论派生和跨平台来源追链。
2. 没有 `seed/support/counterexample/boundary` 的二创链角色，也没有搬运、教程、生成器与真实复刻的区分。
3. 没有模板实体、家族合并规则、四段可迁移语法和独立落库文件。
4. 生命周期只有 topic 的 `hot/periodic/evergreen_search`，不能表达 `query_candidate → observed → replicated → rising/mature/fatigued/evergreen_carrier`。
5. 生成链只能绑定通用机制和风格，不能记录模板版本、刷新时间、固定/替换槽位、支持/反例与素材门。

因此修改前只能输出 `needs_research` 与通用调研计划，无法生产级回答“这套爆款模板是怎么搜出来、怎样判断还能不能跟、以后怎样调用”。

## 本次最小补丁目标

- 主 Skill 增加趋势任务路由和生成硬门。
- 新增一份完整 radar 参考方法，不把当期模板名单固化为永久公式。
- 新增逐帖 sample CSV 与候选卡 JSONL 合同。
- 保留公开代理、版权、真实性、素材与生命周期边界。
