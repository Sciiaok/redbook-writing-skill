# 跨平台获客、站内承接与自有资产

采集日期：2026-07-16。适用对象：女性向成人亲密关系内容与可能的成人用品商业化。所有平台功能与准入只按本轮可见页面记录，执行前仍需用具体主体、SKU、素材和链路复核。

## 一、先纠正目标

“做了账号以后去哪里导流”至少包含四种完全不同的动作：

```text
external_to_xhs
外部需求场 → 原生内容 → 同一公开身份/系列形成记忆 → 用户主动搜索或访问小红书

xhs_to_native_conversion
小红书内容 → 主页与证据系列 → 经审核的店铺/商品/咨询/留资/私信 → 订单

xhs_to_approved_external
小红书内容 → 经精确页面/工单确认的官方外跳产品 → 其他平台购买（本轮默认blocked）

owned_retention
合法自愿订阅/交易 → 自有站、微信或邮件 → 归档、售后、复购与退出
```

不能把这四条都叫“导流”。第一条是获客与品牌发现；第二条是站内商业承接；第三条必须有官方产品和当前权限；第四条只服务已经合法获得并可退出的用户关系。不存在第五条捷径：二维码、联系方式变体、“看主页/dd/看置顶”、竞品评论区招揽和主动陌生私信都是当前治理风险。

对这个账号，更合理的一级目标不是“跨 App 跳转量”，而是：目标用户在每个平台都能得到原生价值，逐步形成同一名称/系列的记忆；需要进一步证据时主动访问小红书；购买或咨询走经过审核的路径。

## 二、渠道角色图

```text
知乎（搜索型问题）────────────┐
B站/播客（解释力与深度信任）──┤
抖音（广泛发现与命题测试）────┤→ 同一公开品牌/系列名 → 小红书证据系列
线下活动（体验与自愿UGC）─────┘                         ↓
                                                经审SKU/原生组件 → 订单

微信/邮件/自有站：只承接已自愿订阅者的归档、售后和复购，不承担从小红书暗中截流
```

完整结构化矩阵见 [`../acquisition-channels.csv`](../acquisition-channels.csv)。商品资格按surface拆在 [`../sku-registry.csv`](../sku-registry.csv)，非SKU编辑/订阅/合作/活动拆在 [`../offer-registry.csv`](../offer-registry.csv)；任何一条渠道只有在所引用的全部资格记录通过后才允许对应CTA。

### P0：先把小红书自己做成可承接的主页

不要先开五个平台。主页至少要有三个可反复验证的内容系列：

1. 关系沟通：边界、同意、需求表达、关系修复。
2. 身体常识：只做有来源、非诊断、非功效夸大的教育内容。
3. 产品决策：材质、清洁、适用/不适用、隐私包装、售后与选择条件；在具体 SKU 未确认前不放购买/咨询 CTA。

`OFF-017`、`OFF-021`、`OFF-022` 能分别确认搜索信息流、蒲公英创作者合作，以及帮助中心里的客资收集/私信咨询入口。其他组件与外跳能力尚未逐项取得精确当前页面；任何产品入口存在都不等于成人 SKU 自动获准或投放有效。

### P1：知乎 + B站或播客（二选一）

知乎用于捕获已经形成问题的人。适合做“边界怎么谈”“亲密后不适如何沟通与就医”“第一次选产品先看哪些条件”等任务型问答。`OFF-024` 鼓励细分、可信来源和信息增量；`OFF-025` 把伪造经历、露骨低俗和垃圾广告列为风险。默认 CTA 是继续阅读本平台系列/关注，不把回答写成跳转小红书的广告。

B站或播客用于建立解释力和人物信任：非露骨讲解、经授权匿名/多案例合成的关系案例、与有真实资质者的访谈。B站商业合作使用 `OFF-027/OFF-028` 的花火路径，且遵守 `OFF-029` 的色情低俗、伪装身份和垃圾促销边界。播客平台也需按其当前规则逐一复核，本轮未把任一播客平台列为永久安全渠道。

### P2：有稳定产能后才加抖音

抖音用于快速测试一个命题是否能被理解，例如一个沟通误区、一个非露骨场景、一项产品判断条件；不是复制小红书轮播，更不是把用户赶去另一个 App。`OFF-030` 是通用内容底线；`OFF-031/OFF-032/OFF-033` 共同说明“存在情趣用品类目”与“具体 SKU/画面可售”是两回事；商业合作优先 `OFF-034` 巨量星图或抖店原生闭环。

### P3：自有网站、微信与邮件负责留存

自有站可以公开、明确地链接到官方小红书主页，并直接统计按钮点击。微信/邮件只服务已经自愿订阅的人，承担内容归档、售后、复购教育和政策变化后的用户关系，不通过小红书暗号获取名单。`OFF-035` 证明腾讯广告/企业微信/CRM有官方线索承接能力，不代表从小红书违规导微信被允许。

亲密关系投稿、健康问题、聊天记录和订单可能包含敏感个人信息。`LEG-003` 要求特定目的、充分必要、严格保护和通常的单独同意；账户至少要有授权/撤回、最小收集、去标识、访问权限、保留期限和删除机制。

### 补充：合作与线下体验

- 合作对象优先女性健康科普、关系教育、生活方式创作者；各平台用蒲公英、花火、星图等官方合作路径，披露关系，不让合作方假装普通用户。
- 线下沙龙、展览、读书会或匿名问题墙可承担高信任体验。参与者是否拍摄/发布完全自愿，不奖励打卡、不强制发帖；活动同时做好年龄、隐私、拍摄同意和危机求助预案。

## 三、现有成人品类案例能证明什么

| 案例 | 可观察结构 | 不能证明 |
|---|---|---|
| `IND-062` 大人糖多渠道 | 品牌自述使用小红书、播客、微信、微博及线下，表达聚焦性自主/关系议题 | 哪个渠道带来增量、CAC、跨平台转化 |
| `IND-063` 杰士邦女性人群 | 申奖案例展示小红书内容、选品、直播的站内闭环 | 新小号可复刻；结果经独立审计；外站导入有效 |
| `IND-064` 大人糖线下店 | 展览/教育体验可能触发自愿UGC与品牌讨论 | 偶发走红可复制；线下直接导致订单 |
| `IND-065` 多平台传播风险 | 清新审美与女性自主叙事仍可能受商业审核限制 | 2021观察可直接当2026平台规则 |

本轮没有找到带可核验后台的“女性向成人用品：外站 → 小红书 → 成交”成功案例。这个负结果本身重要：不应使用服务商口述的跳转率或订单数制定预算。

## 四、商业承接优先顺序

```text
1. 具体 `shop` surface 与SKU均确认后，使用经审店铺/商品组件
2. 具体 `leadgen`/`dm_commercial` surface、表单同意和SKU均确认后，使用获准客资或私信能力
3. `approved_external_destination` 当前保持blocked；补齐精确官方URL、产品名、日期、账户/行业/SKU和目的地工单后才评估
4. 任一必要资格缺失：停止商业CTA，继续积累非商业内容与证据
```

`OFF-004` 的专业号规则同页出现“性玩具禁入”和“情趣用品/计生用品普通准入”，不能把措辞冲突当漏洞。应拿品牌、型号、材质、形态、功能、包装、图文素材、资质和拟用链路向平台人工确认，并保存答复日期/工单。`OFF-005` 同时限制模板/虚构故事营销、间接导流、第三方平台和两性生活/拟态物等表达；内容通过不等于商品、广告或落地页也通过。

商业内容还需过法律层：

- `LEG-004`：知识、体验、测评附购买方式时需具广告可识别性，不得欺骗点击。
- `LEG-005`：性能、用途和效果必须真实；不得以非医疗商品混淆疾病治疗或伤害未成年人。
- `LEG-001`：普通评价与自行/受托推销要按真实目的和利益关系区分。

## 五、指标与归因

### external_to_xhs

```text
外站曝光/完读
→ 自有链接点击（只有自有站等允许且可测的场景才精确）
→ 小红书品牌词/系列词搜索的方向性变化
→ 主页访问
→ 新关注
→ 7/30日回访
```

跨平台通常没有用户级闭环。不能给精确转化率时，使用固定系列名、渠道错峰、前后基线和未投放主题作对照，并把结果标为 `directional`。先确认目标指标在当前账号真实可取，再根据基线量预注册观察窗口、最小事件数和决策规则；不机械套“14天+14天”或其他固定周期。

### xhs_to_native_conversion

```text
内容消费 → 商品/组件点击 → 加购/线索 → 支付 → 退款 → 30/90日复购
```

线索类额外记录：提交、有效线索、是否明确同意联系、成交。最终判断使用贡献毛利、CAC、LTV、退款与投诉，不把粉丝或点赞当商业结果。合作和付费优先采用蒲公英、花火、星图等原生后台指标。

## 六、用户尚未考虑的生产与经营门槛

1. **具体 SKU**：名称、形态、材质、功能、资质与禁售项，而不是笼统“成人玩具”。
2. **身份透明**：编辑内容与商业线可以分轨，但公开关联；不能用独立素人号假测评。
3. **年龄与危机安全**：未成年人隔离、胁迫/暴力/创伤求助、性骚扰评论和紧急资源预案。
4. **投稿治理**：授权范围、是否可改编、撤回、去身份化、聊天截图权属和删除机制。
5. **产品证据**：材质、清洁、禁忌、说明书、适用/不适用、非医疗边界。
6. **成交体验**：私密包装、卫生用品退换规则、售后响应、投诉与召回；这些比内容流量更直接影响口碑和复购。
7. **渠道集中风险**：保留自有域名、合法订阅名单、素材版权、来源账本和数据备份。
8. **生产能力**：每周一个母题，再按平台重新组织，不复制粘贴；先稳定两个渠道，再扩第三个。
9. **归因设计**：每次只新增一个渠道/合作变量，否则无法知道增长来自哪里。

## 七、写入 Skill 的硬校验

- `direction` 不得为空，不能输出一个笼统“导流方案”。
- `permitted_cta` 必须有当前官方依据，否则标 `needs_current_platform_check`。
- 每个商业CTA必须引用 `sku-registry.csv` 的具体 `eligibility_id + surface`；所有必要记录均confirmed才可生成，不能用一个总的SKU状态放行全部入口。
- 外跳能力缺精确URL、产品名、日期、账户/行业/SKU和目的地工单时保持blocked。
- 外部到小红书无用户级归因时，只能标 `directional`，不得给精确转化率。
- `public_identity` 一致且透明；投稿、综合改编、虚构演练必须标来源类型。
- 禁止生成二维码、微信号、暗号、看主页/dd/看置顶、多账号伪装、评论截流和自动私信方案。
- 渠道矩阵必须同时有主指标、归因方法、合规作用域、负责人、基线窗口和测试状态。
