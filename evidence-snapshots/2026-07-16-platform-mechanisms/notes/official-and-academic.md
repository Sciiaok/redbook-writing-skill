## TECH-002｜小红书搜索相关性工程论文（KDD 2026）

- 标题：Optimizing Generative Ranking Relevance via Reinforcement Learning in Xiaohongshu Search。
- URL：https://arxiv.org/html/2512.00968v2
- 作者与身份：14 名作者，多名来自 Xiaohongshu Inc.；KDD 2026 ADS Track 已接收。
- 数据与方法：RANDOM 与 LONGTAIL 测试集各 15,000 组人工标注 query-note；训练数据为 50,000 个查询×20 篇笔记，共 100 万组，由 25 名专业标注员历时三个月标注；另有在线 A/B 测试。
- 与内容生产直接相关的原始机制：
  - 相关性是 -1/0/1/2/3 五级，不是“有关键词/没关键词”的二元命中。
  - 人工准则考虑搜索意图一致、主题一致、预期用户参与等维度。
  - 模型第一步会做查询分析、类目/分类、关键词匹配、相关内容占比等高层语义比较；第二步再应用细分场景规则限制相关性上限。
  - 长尾查询定义为单日 PV 少于 5 的查询；平台专门评估其语义理解能力。
- Skill 影响：关键词应该服务于完整意图匹配；标题、封面、正文只“出现词”而主体偏题，会被内容占比和语义一致性问题拖累。选题库要同时覆盖主流词和高意图长尾词。
- 不能支持：论文讨论相关性模块，并未公开完整搜索最终排序的所有质量、时效、个性化、互动或安全特征，更没有给创作者一个可逆向的权重公式。
- 证据等级：A。

## ACA-003｜标签再挪用与受众控制研究

- URL：https://arxiv.org/abs/2501.18210
- 方法：分析 5,800 篇帖子，访谈 24 名活跃用户。
- 发现：部分女性用户使用与内容字面无关、预计男性不感兴趣的标签，试图减少不想要的男性受众进入。
- 局限：能证明实践、动机和用户感受存在，不能证明标签对推荐分发有稳定因果效果；也可能造成搜索污染和内容—标签不匹配。
- Skill 影响：作为“安全受众需求”的研究输入，而不是运营技巧。账号应通过明确受众语言、问题选择和合规内容建立女性向语境，不使用无关标签操纵分发。
- 证据等级：B。

## 首轮交叉核验结果

1. OFF-001 确认“内容数据+互动行为”是推荐输入类型，但没有公布权重。
2. ACA-002 出现精确 CES 公式，却无法给出原始技术来源。因此它不能填补 OFF-001 没披露的部分。
3. OFF-003 证明平台产品存在丰富的搜索入口和查询来源，但没有公布搜索排序公式。
4. 成人商品、广告、商业笔记、自然笔记是四个不同规则面；任何单一准入页面都不能替代另外三个面的核验。
5. OFF-007 与 OFF-005 对“假装真人真实经历+情色聊天/网文+产品导流”的风险形成交叉证据；该方向需要改造成明确虚构的非情色叙事，或真实匿名案例的教育性拆解，且不能伪造产品体验。
6. TECH-002 和 TECH-003 提供了比“CES 公式”更接近生产系统的机制：搜索重视 query-note 的语义、类目、关键词与内容占比；新笔记冷启动会利用图文内容信号，推荐评估同时观察点击、阅读/时长和互动。

### TECH-004｜Qilin 多模态检索会话数据集

- 原始论文：https://arxiv.org/abs/2503.00501

### TECH-005 / TECH-006｜NoteLLM 与 NoteLLM-2

- 原文：https://arxiv.org/abs/2403.01744 、https://arxiv.org/abs/2405.16789
- 场景：小红书 I2I 相关笔记召回/表征，不是首页最终排序。
- 内容字段：标题、正文、话题标签、类目；NoteLLM-2 再加入图像。NoteLLM 用“用户先看 A 再点击 B”的共现构造相关笔记对。
- Skill 影响：标题、正文、标签、类目与图像应表达一致主题；不能把“平台理解标签”改写成“多塞标签就有额外权重”。
- 证据等级：A。

### ACA-004｜The Informal Labor of Content Creators

- 原始论文：https://arxiv.org/abs/2409.08360
- 方法：九个月数字民族志。
- 发现：KOC 商业劳动常被包装成普通自发分享，帮助理解平台为何强调经营身份透明与反对伪素人营销。
- 局限：不能证明单篇处罚、推荐权重或转化效果。
- Skill 影响：编辑内容与商业内容分轨；利益关系、素材来源和真实体验均需披露。
- 证据等级：B。

## 第二轮常见传言审计

| 说法 | 证据判定 | Skill 中的正确写法 |
|---|---|---|
| 每篇先给 200/500/1000 人，再逐层流量池 | D：未找到官方/生产论文依据 | 平台有新内容冷启动问题，也有内容表征方案；未公开固定曝光池或阈值。 |
| CES 是点赞/收藏/评论/分享/关注的固定加权公式 | D：未验证 | 互动是公开信号类型，但普通信息流权重未披露。 |
| 收藏一定比评论/点赞权重高 | D | 只能按账号、类目、入口做实验，不给通用权重。 |
| 连续发 7 天，账号就会获得垂类标签 | D/概念混淆 | 已确认内容 taxonomy、用户兴趣和风险/真诚治理；未确认静态创作者垂类标签决定全部分发。 |
| 晚上 8 点是全站最佳发布时间 | D | 时间是上下文；统一最佳时段未获证实，只能用账号自身数据实验。 |
| 违规后统一限流 7/15/30 天 | 错误泛化 | 不同规则有不同处置和作用域；以具体规则、站内信和可见性异常为准。 |
| 蒲公英健康等级异常会拖累主站自然流量 | A 级官方反证 | 该等级影响蒲公英权限/曝光，不影响 App 主站推荐。 |
| 真粉连续点赞会伤号 | A 级官方反证 | 正常喜爱互动无此结论；异常账号、人工干预、互赞买量才是风险。 |
| 多加热门标签就能推给精准人群 | D | 标签是语义线索，不提供受众保证；不相关标签可能破坏相关性。 |
| 新号必须养号、互粉互赞 | 无依据且高风险 | 不把养号当机制；互粉互赞、利益诱导和刷量是明确治理对象。 |
| 低流量就是被限流 | 不成立 | 相关性、供需竞争、点击、消费、互动、审核都可能影响表现；需证据诊断。 |
