# Redbook Writing Skill

面向小红书账号研究、内容生产与复盘的 Codex Skill。它不把“爆款公式”当平台事实，而是把公开来源、近期样本、账号基线、反例、选题、成稿和商业资格串成一条可复核链路。

## 能做什么

- 调研类目、帖子与账号，并保留普通基线、高表现样本和反例。
- 区分官方事实、实操经验、待验证假设、冲突证据与未知项。
- 建立选题库、标题/封面候选、完整成稿和单变量复盘计划。
- 审核虚构故事、聊天记录、真人授权、商业披露和敏感内容。
- 拆解评论参与、站内承接、官方外跳与自有留存，阻断竞品截流和绕审。
- 对 SKU、offer、平台、账号、surface 与具体素材逐项核验商业资格。

## 安装

把 [`redbook-writing`](redbook-writing/) 目录复制到你的 Codex skills 目录，然后在任务中要求使用 `redbook-writing`。

```text
~/.codex/skills/redbook-writing/
```

Skill 会按任务选择 `mechanism`、`discovery`、`refresh` 或 `draft` 模式，并只加载当前需要的参考文件。

## 推荐输入

最有价值的输入依次是：

1. 账号后台数据、历史笔记和当前内容库。
2. 目标用户的具体处境与这篇内容要完成的唯一任务。
3. 喜欢的 3–5 段表达、不喜欢的 2–3 段表达。
4. 出镜条件、真实素材、单篇生产成本与审核时延。
5. 商品型号、说明书、资质、真实体验和不可宣称事项。
6. 真人材料的成年确认、授权、脱敏、撤回与商业使用范围。

没有可核样本时，Skill 只登记研究问题和查询候选，不会用想象补出账号榜单、爆款选题或成稿。

## 目录

```text
redbook-writing/                         # 可安装 Skill
├── SKILL.md                             # 路由与总方法
├── references/                          # 调研、机制、规则、写作与渠道方法
├── assets/                              # 可复制的运行与账本模板
├── scripts/validate_run.py              # 严格落库验证器
└── agents/openai.yaml                   # Skill 元数据
evidence-snapshots/                      # 本次方法研究及来源快照
tests/                                   # schema、验证器与对抗前向评测
```

## 验证

验证一个已经落库的研究运行：

```bash
python3 redbook-writing/scripts/validate_run.py <run-dir> --strict
```

运行仓库测试：

```bash
python3 -m unittest discover -s tests -p 'test_*.py' -v
```

`--strict` 通过只说明数据合同和关键安全门闭合，不代表传播效果得到保证，也不能替代发布时的平台当前规则与人工复核。

## 关键边界

- 不伪造真人故事、聊天记录、测评、订单或社会证明。
- 不生成竞品评论区截流、陌生私信、暗号、联系方式变体或多账号话术。
- 不把固定 CES、初始流量池、标题字数、发布频率或转化率包装成平台规律。
- 敏感内容默认可能被未成年人看到；“18+”标签不是安全隔离。
- 规则、授权或商业资格不清时降级为 `unknown` / `blocked`，不先写 CTA 再补审批。

本仓库不提供法律意见，也不代表小红书或其他平台作出审核、准入或效果保证。
